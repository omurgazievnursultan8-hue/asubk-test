### Фаза 10 — Информация о ведомстве: предложения по разработке (для команды разработки)
_Модуль ТЗ «Модуль управления информацией о ведомстве». Живого экрана в приложении нет —
спроектирован с нуля. Мокап: `mockups/vedomstvo/vedomstvo.html` (8 вкладок, роли
Администратор / Наблюдатель, срез «на дату»). Нумерация `P10-R*` — отдельный счётчик
фазы. Дефектов нет (нечего проверять на живом экране), записей по фазе 10 в
`notes/qa-findings.md` не будет._
- [ ] P10-R1 🔴 Сущность «Ведомство» — singleton + разделение блоков на собственные и зеркала
  - модель: одна запись на систему; создание и удаление недоступны by design, есть только чтение и редактирование;
  - собственные блоки: общая информация (1:1 — полное/краткое наименование, описание, логотип), регистрационные данные (1:1), хронология (1:N, сортировка по дате убыв.), банковские счета (1:N), документы (1:N), контакты (1:N по типам);
  - зеркала: оргструктура и показатели — только чтение, источник в чужих модулях (P10-R5, P10-R6);
  - наименование только на русском (двуязычие кыр/рус не вводится).
- [ ] P10-R2 🔴 Банковские счета — append-only + инвариант «основного» счёта
  - модель: счёт **не удаляется**, а закрывается датой (`closedAt`) — реквизит из прошлого платёжного поручения должен оставаться разрешимым;
  - инвариант: не более одного «основного» счёта **в разрезе валюты**, считается **только по действующим** счетам (закрытый основным быть не может);
  - при закрытии счёта флаг «основной» снимается автоматически; нарушение инварианта — блокирующая ошибка, а не предупреждение;
  - в UI — фильтр «Действующие / Все» и копирование реквизитов счёта в буфер.
- [ ] P10-R3 🔴 Значения полей — из справочников, free-text запрещён
  - модель: банк (БИК принадлежит банку, а не нашему счёту — не вводить отдельным полем), валюта, назначение счёта, орган регистрации, тип телефона, тип e-mail, тип документа — ссылки в модуль справочников;
  - причина: free-text даёт расхождение значений между модулями («ОАО «РСК Банк»» / «РСК банк» / «RSK») и ни один отчёт по ним не сходится.
- [ ] P10-R4 🟠 Документы ведомства + производный статус срока действия
  - модель: тип (устав / свидетельство / лицензия / разрешение / постановление), номер, дата выдачи, срок действия (может быть бессрочным), выдавший орган, файл (вложение подсистемы документооборота);
  - статус **вычисляется, не хранится**: Бессрочный · Действует до … · Истекает через N дн. (порог 90) · Истёк;
  - сводка по вкладке: просроченные / истекающие документы — отдельным индикатором.
- [ ] P10-R5 🟠 Оргструктура на вкладке — зеркало, резолвится на дату просмотра
  - модель: узлы версионны (создание, переименование, ликвидация; смена руководителя) — витрина показывает срез на выбранную дату, а не текущее состояние;
  - редактирование — только в модуле оргструктуры, здесь read-only ссылка, не копия;
  - глубина — до подразделений, без персоналий: ФИО руководителя — публичный атрибут узла, а не ссылка на карточку сотрудника; вакансия — валидное состояние.
- [ ] P10-R6 🟠 Показатели — зафиксированные срезы, а не расчёт на лету
  - модель: агрегаты считаются батчем в нерабочее время и хранятся срезами с датой расчёта; витрина берёт последний срез на дату ≤ выбранной и показывает дату расчёта;
  - только всеорганизационные агрегаты (портфель, выдано с начала года, доля просрочки, число заёмщиков) — без разрезов по подразделениям;
  - дельта — к предыдущему срезу; доля просрочки меняется в **процентных пунктах**, не в процентах от процента.
