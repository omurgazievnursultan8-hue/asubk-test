## 2. Модель состояния (§3)

Только факты. Ни одного производного поля. Коммент-владелец у каждого массива.

**Владеет этот модуль (правится):**
```
SUBJECTS       {inn,name,personKind:'юр'|'ИП'|'физ',legalForm,ownershipForm,industry,
                doc{kind,no,date},addrLegal,addrFact,regDate}
SUBJECT_EVENTS {inn,kind,date,basis,doc,successorInn?}          append-only   ← Р-9
CHANNELS       {inn,kind:'тел'|'email'|'адрес корресп.',value,from,closedAt?}  ← Р-10
BANK_REQ       {inn,bank,account,currency,main,from,closedAt?}  append-only    ← Р-10
CATEGORY_LOG   {inn|creditId,level,from,reason,source}          append-only
GROUP_LOG      {inn,group,from,basis,confirmedBy}               append-only
COMMITTEE_REFS {id,no,date,kind:'КБК'|'КРР',subject}
```

**Зеркала — read-only, правка = переход в модуль-источник (И-5):**
```
CREDITS    {id,no,inn,amount,balance,currency,overdueDays,debt{principal,interest,penalty,fees,costs},
            program,kind,closedAt?}                              ← кредитный модуль
FACTORS    {creditId,level:'mid'|'high',type,committeeRef,from,to?}  ← комитет
EVENTS_RAW {id,inn,creditId?,type,detectedAt,source:'zalog'|'payments'|'collection'|'acts',
            wouldGive:'mid'|'high',payload,committeeRef?:null,dismissedAt?}  ← Р-5 (ОВ-6: seeded)
OVERLAYS   {creditId,kind:'suppress181',from,deadline,sourceRef}     ← restructuring
PROCESSES  {id,credits:[creditId],procedure,phase,confirmed,owner,region}  ← collection (M:N)
ASSIGN     {objectKind:'кредит'|'предмет'|'заявка',objectId,role,empId,from,to?,source}  ← kuratorstvo
CONFLICTS  {id,empId,inn,trigger,declaredAt,svbAt,boardNoticeAt?,decision?{kind,date,doc},closedAt?}  ← Р-6
PLEDGE_IX  {creditId,index,required,insured,itemsByRegion:[]}    ← zalog
ACTS       {id,inn,creditId?,kind:'годовой'|'полное погашение',dueDate,signedAt?,status}
DOCS       {id,inn,scope:{kind,ref},type,date,ownerModule,file}     ← Р-8
CHECKS_EXT {inn,source:'КИБ Ишеним'|'чёрный список'|'скоринг',takenAt,appId,payload}  ← Р-8
RELATED    {inn,personId,fio,role,share?,from,to?}                  ← реестр лиц (Р-7)
WORKDAYS   производственный календарь КР (пн-пт − праздники, ОВ-2 — один массив)
```

### Поток данных — категория (ядро, Р-3/Р-5)

```
дни просрочки ─┐
                ├─ catOfCredit → worst-of(daysEff, FACTORS с committeeRef) ─┐
OVERLAY 181 ───┘  подавление ДО worst-of                                    ├─ catOfBorrower =
EVENTS_RAW ─── committeeQueue: сигнал, категорию НЕ двигает (И-1) ──────────┘  worst-of по действующим,
                     ↑ ставят zalog / payments / collection / acts             null если кредитов нет (И-3)
```

Подавление 181-го применяется **на уровне кредита, до worst-of** (иначе оверлей по одному
кредиту глушил бы риск по остальным). Событие само по себе категорию не двигает — между
событием и категорией лежит решение комитета (FACTORS с непустым `committeeRef`).

## 3. Чистые функции — контракт модуля (§4)

```
КАТЕГОРИЯ   catByDays(days)         ≤5→low · 6..180→mid · >180→high            (п.12)
            isSuppressed181(cr,d)   raw==='high' && days>180 && активный OVERLAY на дату
            catOfCredit(id,d)       → {days,raw,suppressed,daysEff,factors[],level}
                                       worst-of(daysEff, факторы С РЕШЕНИЕМ КОМИТЕТА)
            catOfBorrower(inn,d)    worst-of по НЕзакрытым; null если действующих нет
ГРУППА      groupOf(inn,d)          лестница-доминирование (НЕ ординальная шкала):
                                       Блок1 (субъект/признание): 3.1·3.2·4·5
                                       Блок2 (кредит в процедуре): max(2.1<2.2<2.3)
                                       иначе 1.1 · ТОЛЬКО подтверждённая процедура (И-2)
ДОЛГ        totalDebt(inn)          Σ по действующим, пять статей §9
            overdueDebt(inn)        Σ просроченных частей
            coverageOf(inn)         зеркало zalog: worst и агрегат index vs required
ОБЯЗАТ-ВА   obligations(inn,d)      → [{code:'О-1'|'О-2'|'О-3'|'О-4',basis,dueDate,status}]
                                       status: 'не наступило'|'в срок'|'просрочено'|'исполнено'
            addWorkdays(date,n)     по WORKDAYS
КОМИТЕТ     committeeQueue(inn,d)   EVENTS_RAW без committeeRef и без dismissedAt
                                       → [{type,source,detectedAt,wouldGive,waitingDays}]
КОНФЛИКТ    conflictState(c,d)      → {phase:'заявлен'|'уведомление'|'на рассмотрении'|'урегулирован',
                                        noticeOverdue,suspendedFrom,suspendedTo|null,outcome}
            suspendedEmployees(inn,d)  все отстранённые от ИНН на дату
СУБЪЕКТ     subjectState(inn,d)     последнее SUBJECT_EVENTS на дату
            isReadOnly(inn,d)       true при 'долг переведён' | 'ликвидирован'
КУРАТОРСТВО curatorMatrix(inn,d)    объект × роль на дату через ASSIGN (Ш-3)
```

### Инварианты, проверяемые в рантайме (рендер их использует, не только тесты)

- **И-1** категорию меняет только `FACTORS` с непустым `committeeRef` либо дни просрочки; `EVENTS_RAW` — никогда.
- **И-2** группу двигает только `PROCESSES.confirmed === true` либо терминальный исход.
- **И-3** у заёмщика без действующих кредитов `catOfBorrower === null` → плитка «не применяется».
- **И-4** отстранение по конфликту действует на **все** кредиты ИНН, не на один.
- **И-5** зеркала не редактируются — ни одного `input`/`select` внутри зеркальных секций, только ссылка «открыть в …».
