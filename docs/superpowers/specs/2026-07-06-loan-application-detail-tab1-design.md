# Дизайн: детальная заявки — перестройка первой вкладки + вынос условий в отдельную вкладку

**Дата:** 2026-07-06
**Файл-мокап:** `mockups/loan-application/loan-application.html` — детальная заявки,
вкладка `#tab-0` («Общая информация») + новая вкладка «Условия кредита»
**Route (живая система):** `/loan-applications/{id}` → первая вкладка «Общая информация»
**Связанные дефекты:** P2-01 (аффорданс полей), P3-15 (пустые select одобренных условий),
P3-17 (номер в шапке ≠ id, 5 статус-полей дублируются на всех вкладках)

## Проблема

Сейчас первая вкладка «Общая информация» = три серые колонки read-only:
1. Данные заявителя (ИНН, телефон, адрес);
2. Условия кредита заёмщика (запрошенные сумма/срок, программа, льготные, даты, источник);
3. Условия кредита, одобренные комиссией (сумма+диапазон, срок+диапазон, ставка, резерв,
   периодичность, метод, день платежа, программа, льготные, источник, комментарий).

Дефекты этого расклада:
- **Смешаны роли и жизненные циклы.** Паспортные данные заявителя (статичные) лежат рядом с
  редактируемыми условиями (черновик → комиссия → замок). Всё выглядит одинаково серым — корень P2-01.
- **Две колонки условий рядом (запрошено vs одобрено)** путают: часть полей одобренной колонки
  (ставка/резерв/периодичность/метод/день) вообще не имеет «запрошенного» аналога — сравнивать не с чем.
- **Условий будет больше** (полный набор 15+ полей) — на первой вкладке рядом с заявителем не читаемо.

## Жизненный цикл условий (бизнес-сценарий)

Кредитный специалист редактирует условия → отправляет комиссии на рассмотрение → комиссия вносит
свои изменения и выносит окончательное решение → после решения условия менять нельзя.

| Статус заявки | Кто редактирует условия | Какие поля редактируемы |
|---|---|---|
| Черновик | кредитный специалист | запрос: сумма, срок, программа, льготные, источник — белые |
| На рассмотрении комиссии | комиссия | + операционные: ставка, % резерва, периодичность, метод, день платежа, обработка выходных — белые |
| Одобрено / Отклонено | никто | всё серое read-only, diff зафиксирован |

## Решение

### ① Первая вкладка «Общая информация» → сводка (вариант C)

Раскладка — **2 колонки**, снизу одно поле на всю ширину:

- **Колонка 1 «Данные заявителя»** — ФИО/Субъект, номер телефона, адрес. Паспортная карточка;
  редактируется только телефон в режиме edit, остальное read-only.
- **Колонка 2 «Статус заявки»** — вертикальный таймлайн: `Черновик → На рассмотрении комиссии →
  Одобрено` (ветка `Отклонено` — красная веха решения). Активный шаг подсвечен; пройденные — с
  датой и автором перехода. Цифры условий (сумма/срок/ставка) сюда **не** дублируются.
- **Снизу на всю ширину** — «Дополнительная информация» (одно длинное поле; textarea в режиме edit,
  иначе read-only текст).

**Шапочный горизонтальный stepper «Статус заявки» (над лентой вкладок) удалён** — он дублировал
жизненный цикл. Единственный источник статуса теперь — таймлайн во 2-й колонке.

Цель: первая вкладка отвечает на «кто заявитель» и «где заявка в процессе», а не вываливает 3 простыни.

### ② Новая вкладка «Условия кредита» → полный набор, одна колонка, инлайн-diff (варианты A+A)

Полный набор условий в **одной колонке**. Модель отображения:

- **Пересекающиеся поля** (у заёмщика есть запрос, комиссия может изменить): сумма, срок,
  программа, льготные периоды, источник финансирования.
  - комиссия изменила → показываем `запрошено` (зачёркнуто) `→ одобрено`;
  - не изменила → одно значение.
- **Операционные поля** (только комиссия/программа, «запроса» нет): годовая ставка, % резерва,
  периодичность платежей, метод погашения, день платежа, обработка выходных.
  - просто значение + справка «из программы».

**Аффорданс полей (чинит P2-01)** — переиспользуем design-system компонент P2-R1:
- обязательное-редактируемое → белый фон + рамка + красная `*`;
- необязательное-редактируемое → белый фон + рамка;
- read-only → серый фон, без рамки.

Редактируемость полей определяется статусом заявки по таблице жизненного цикла выше.

### Шапка (заметка, отдельная задача)

5 статус-полей дублируются на всех вкладках (P3-17). Таймлайн на первой вкладке частично их
заменяет — шапку можно сжать отдельной задачей; в объём этой перестройки не входит.

- `mockups/loan-application/loan-application.html`:
  - перестроить `#tab-0`: убрать 3 колонки условий → оставить «Данные заявителя» + вертикальный таймлайн;
  - добавить вкладку «Условия кредита»: одна колонка, инлайн-diff, аффорданс P2-R1, замок по статусу;
  - обновить `TABS`/`renderTabbar()` — новая вкладка в ленте.
