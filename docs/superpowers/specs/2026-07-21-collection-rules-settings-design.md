# Дизайн: страница настройки правил доступности мер (макет «Взыскание»)

**Дата:** 2026-07-21
**Файл макета:** `mockups/collection/collection.html` (единый self-contained HTML/JS)
**Проверка:** `node scripts/inspect/collection-check.mjs` (jsdom-харнесс)

## Проблема

Правила «когда мера доступна» и «кому мера доступна» зашиты в код константами:

- **Кому (В-9):** `MEASURE_SUBDIV` (явное подразделение на вид меры) + `SECTION_SUBDIV`
  (дефолт по разделу) × `ROLE_SUBDIV` (роль → подразделение). Читается в `subdivReason`.
- **Когда — стадия:** `SECTION_CLEVEL` (раздел → минимальная ступень эскалации) +
  `CONTOUR_LEVEL`. Читается в `sequenceReason`.
- **Когда — последовательность:** порядок вех в `CONTOURS[c].phases` + `MILESTONE_PHASE`.
  Читается в `sequenceReason`.
- **Когда — гейты:** `GATES` (мера → нужно решение комитета / поручение Председателя).
  Читается в `gateReason`.

Правила невидимы для заказчика и приёмки, их нельзя проиграть «а что если». Нужна
страница, где эти четыре оси видны и редактируемы прямо в макете, а список мер у
процессов сразу реагирует на изменения.

## Цель

Отдельный экран «Настройки» в макете с живым редактированием всех четырёх осей правил.
Демонстрирует заказчику будущий админ-экран реального приложения. Изменения переживают F5
(демо-персист), сбрасываются к дефолту в один клик.

## Не-цель (YAGNI)

- Не добавляем/не удаляем сами фазы, разделы, роли или виды мер — только правим правила
  между существующими сущностями. Вкладка «Фазы» — **только переупорядочивание** вех
  внутри контура, без добавления/удаления.
- Не редактируем расчётные формулы долга, шаблоны документов, справочники результатов.
- Не делаем серверную персистентность/мультипользовательность — это макет, localStorage.

## Подход к данным: единый слой `RULES` (вариант A)

Четыре редактируемые структуры собираются в один мутабельный объект `RULES`. Сид —
глубокая копия текущих констант, замороженная как `RULES_DEFAULTS` (источник для сброса).

```js
const RULES_DEFAULTS = deepFreeze({
  measureSubdiv: {...MEASURE_SUBDIV},     // явное подразделение на вид (перекрывает раздел)
  sectionSubdiv: {...SECTION_SUBDIV},     // дефолт подразделения по разделу
  roleSubdiv:    {...ROLE_SUBDIV},        // роль → подразделение
  sectionClevel: {...SECTION_CLEVEL},     // раздел → минимальная ступень эскалации
  gates:         deepClone(GATES),        // мера → {organ, poruchenie}
  contourPhases: mapValues(CONTOURS, c => [...c.phases]),  // порядок вех в контуре
});
let RULES = deepClone(RULES_DEFAULTS);
```

Гейт-функции (`subdivReason`, `sequenceReason`, `gateReason`) и производные
(`subdivOf`, `MILESTONE_PHASE`-обход, порядок фаз) читают из `RULES.*`, а не из
исходных литералов. Исходные константы остаются как сид `RULES_DEFAULTS` и больше нигде
не читаются напрямую в логике доступности.

**Почему A, а не оверлей/правка констант:** один источник истины для чтения, дефолты для
сброса в наличии, откат без риска сломать сид необратимо.

## Персистентность

- Ключ localStorage `asubk-collection-rules-v1` — **отдельно** от состояния процессов
  (`asubk-collection-state-v1`). Версионный суффикс — инвалидировать при смене схемы правил.
- `persistRules()` — сохраняет `RULES` после каждой правки (симметрично `persistState`).
- `restoreRules()` — в bootstrap перед первым рендером наполняет `RULES` из снимка.
- `?reset` в URL чистит **оба** ключа (процессы и правила).
- Кнопка «Сбросить к дефолту» на каждой вкладке (сброс своей секции `RULES`) + глобальная
  (весь `RULES = deepClone(RULES_DEFAULTS)`).

## Навигация

- Шестерёнка в шапке → hash-view `#settings` (переживает F5, как текущие view через
  `restoreFromHash`).
- Полноэкранная страница, 4 вкладки: **В-9 «Кому» · Стадии · Гейты · Фазы**.
- Возврат — на список процессов (кнопка/крестик, снимает hash).

## Вкладки

### 1. В-9 «Кому»
- Грид: строка = вид меры (48), столбец = подразделение (ДАК/ОД/РП/ДПО/ОПК/САК). Ячейка —
  чекбокс «подразделение может регистрировать этот вид».
- Эффективное значение = `RULES.measureSubdiv[kind]` (если задано явно) иначе
  `RULES.sectionSubdiv[section]`. Правка ячейки пишет в per-kind `measureSubdiv[kind]`
  (материализует раздел-дефолт в явный список при первой правке).
- Снизу мини-таблица `roleSubdiv`: роль → подразделение (селект; `—` = read-only роль).
- **Предупреждение:** если у вида не осталось ни одного подразделения → строка помечается
  бейджем «⚠ никто не сможет зарегистрировать». Не блокирует.

### 2. Стадии «Когда»
- Таблица: раздел (Досудебный/Внесудебный залог/Судебный/…) → селект ступени `0–5`
  (`sectionClevel`). Подсказка: «раздел доступен, когда контур процесса достиг этой ступени».

### 3. Гейты
- Список видов мер, у которых сейчас есть гейт (`GATES`); для каждого — два тумблера:
  «нужно решение комитета (орган)» и «нужно поручение Председателя». Плюс возможность
  добавить гейт к виду без гейта (селект вида → включить тумблер).

### 4. Фазы
- Для каждого контура (К0…К7) — список вех в текущем порядке, кнопки ↑/↓ на элементе.
  Переупорядочивание пишет в `RULES.contourPhases[c]`. Добавление/удаление недоступно.
- Подсказка: «порядок задаёт последовательность-предусловие для вех-мер (§3.1)».

## Поток данных

```
чтение гейта (subdivReason/sequenceReason/gateReason)
        │  читает
        ▼
     RULES.*   ──(нет ключа)──►  дефолт (sectionSubdiv / базовое поведение)

правка ячейки на вкладке
        │
        ▼
  мутация RULES.*  →  persistRules()  →  перерисовка вкладки
        │
        └─►  список мер у процессов при следующем рендере берёт новые правила
             (availableKinds/measureKindOptions уже читают гейты → RULES)
```

## Компоненты (в `collection.html`)

- `RULES_DEFAULTS`, `RULES`, `deepClone`/`deepFreeze` хелперы (рядом с матрицей В-9).
- Рефактор чтения: `subdivOf`, `subdivReason`, `sequenceReason`, `gateReason` и обход
  `MILESTONE_PHASE`/порядка фаз переводятся на `RULES.*`. `MILESTONE_PHASE` остаётся
  производным от `RULES.contourPhases` (пересчёт при правке порядка фаз).
- `panelSettings()` + `renderSettings()` — рендер экрана и вкладок (`settingsTab` — активная).
- Хендлеры правок: `toggleV9(kind, subdiv)`, `setRoleSubdiv(role, subdiv)`,
  `setSectionClevel(section, level)`, `toggleGate(kind, field)`, `movePhase(contour, i, dir)`.
  Каждый: мутирует `RULES`, зовёт `persistRules()`, перерисовывает.
- `persistRules`/`restoreRules`/`resetRulesSection`/`resetRulesAll`.
- Навигация: пункт в шапке + ветка в `restoreFromHash` для `#settings`.

## Обработка ошибок / крайние случаи

- Битый снимок правил в localStorage → `try/catch`, откат к `RULES_DEFAULTS` (как
  `restoreState`).
- Роль `—` (Наблюдатель) в `roleSubdiv` не редактируется (остаётся read-only ролью).
- Ноль подразделений у меры — разрешено, но с бейджем-предупреждением (демонстрирует
  последствие, не запрещает).
- Переупорядочивание фаз, ломающее осмысленную последовательность — допускается (песочница),
  предупреждения нет: это осознанная правка порядка.

## Тестирование (`collection-check.mjs`)

- **Регрессия:** дефолтные `RULES` глубоко идентичны исходным константам
  (`MEASURE_SUBDIV`/`SECTION_SUBDIV`/`ROLE_SUBDIV`/`SECTION_CLEVEL`/`GATES`/порядок фаз) —
  поведение до появления страницы не меняется.
- **Правка В-9:** снятие единственного подразделения у вида → вид исчезает из
  `availableKinds(p)` для соответствующей роли.
- **Правка стадии:** повышение `sectionClevel` раздела → мера раздела блокируется
  `sequenceReason` на процессе низкой ступени.
- **Сброс:** после правки `resetRulesAll()` → `RULES` снова идентичен `RULES_DEFAULTS`.
- Существующие 57 проверок остаются зелёными.
