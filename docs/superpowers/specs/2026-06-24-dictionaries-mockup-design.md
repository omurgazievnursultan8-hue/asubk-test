# Витрина справочников — мокап (design spec)

**Дата:** 2026-06-24 · **Автор:** QA/docs workspace · **Статус:** на согласовании

## 1. Цель

Один self-contained HTML-мокап, показывающий **все справочники кредитного модуля**
(47 из меню + 3 picker-only) как **витрину-каталог**: каждая секция — рабочий
экран-список в целевом (to-be) виде. Файл — **база для других мокапов**: из него
переиспользуются дизайн-токены, грид-разметка, бейджи статусов, picker-движок и
готовые таблицы значений каждого справочника.

Не прод-код. Источник истины по as-is — `requirements/tz/08-spravochniki.md`.

## 2. Решения (согласовано с владельцем)

| Вопрос | Решение |
|---|---|
| Роль файла | Каталог-витрина: одна страница, все справочники подряд |
| Данные/стиль | **To-be** — чистый, дизайн-система; тест-мусор вычищен |
| Секция | Рабочий **экран-список**: тулбар + фильтр + таблица |
| Архетип | **Единый** (рек. §8 #1): у всех 47 тулбар `Создать/Изменить/Просмотр/Удалить` |

## 3. Файл и размещение

- Путь: `mockups/dictionaries/dictionaries.html` (single-file, как
  `mockups/loan-program/loan-program.html`).
- Дизайн-система: те же CSS-переменные `--asubk-*`, скопированные из loan-program
  (шрифт, #006AF5, #192434, бейджи `status-*-bg`, грид-рамка, control-h 36px).
- Без сборки/зависимостей; чистый HTML+CSS+JS, открывается файлом.

## 4. Раскладка

```
┌─ sidebar (анкеры) ──┬─ main ───────────────────────────────┐
│ [фильтр по назв.]   │ topbar: «Справочники» · счётчик 47+3  │
│ ▸ Решения/получатели│ ┌── секция: Статус решения ─────────┐ │
│ ▸ Источн./валюты    │ │ h2 + чип S · slug · «3 строки»     │ │
│ ▸ Ставки            │ │ toolbar [Создать|Изменить|Просмотр │ │
│ ▸ Платежи/расчёты   │ │          |Удалить]  [+ условие]    │ │
│ ▸ Договоры/залог    │ │ ┌ grid ─────────────────────────┐ │ │
│ ▸ Субъекты/заёмщик  │ │ │ Наименование │ ...            │ │ │
│ ▸ Кредит/график     │ │ └───────────────────────────────┘ │ │
│ ▸ Товары/счета      │ └───────────────────────────────────┘ │
│ ▸ Picker-only (3)   │ ... ×50 секций                        │
└─────────────────────┴───────────────────────────────────────┘
```

- **Sidebar** — не CRUD-навигация, а **якоря-оглавление**, сгруппированные по
  доменам §2.1 ТЗ. Клик → скролл к секции (`scrollIntoView`). Активная секция
  подсвечивается по скроллу (IntersectionObserver). Сверху — поле фильтра: прячет
  секции/пункты, чьё название не совпадает.
- **Секция** = карточка экрана:
  - заголовок: `<h2>` название · чип архетипа `S`/`R` · slug-маршрут (моноширинный) ·
    счётчик `«N строк»` (для усечённых — `«показано N из M»`).
  - тулбар: `Создать` (primary) · `Изменить` · `Просмотр` · `Удалить` ·
    `+ Добавить условие поиска`. Кнопки в стиле дизайн-системы.
  - грид: рамка `--asubk-border-base`, заголовок 14px/500, тело 16px, единая
    плотность (рек. §8 #7).

## 5. Колонки и данные

- **Источник:** `.auth/dict/_all.json` поле `cellSample` (row-major: заголовки +
  значения). Парсится/курируется **вручную** per-dict, т.к. пустые ячейки в дампе
  схлопнуты — при переносе восстанавливаются и чистятся (to-be).
- **Архетип `S`** — доменные колонки как на стенде (`Наименование`, либо
  `Процент`, `Период(мес.)`, `Код`+`Наименование` и т.п.).
- **Архетип `R`** (`industryDirections`, `collateralTypes`, `commissions`,
  `creditTerms`, `calculationCoefficients`, `individuals`, `organizations`) —
  колонки `Наименование · Код · Статус · Дата создания` (у реестров субъектов —
  доменные ИНН/ФИО/паспорт, ИНН/ОКПО/адрес).
- **Статусы → бейджи** (рек. §8 #6): `Активный/Действует`→success, и т.д.;
  применяется в колонке «Статус» и в справочниках-статусах.
- **To-be чистка:** убрать `соновное 1`, `123/321`, `hidshgiu`, `шрафы`,
  `Aibek rate`, англ. заголовки `agreement-template*`/`destination-accounts`
  → локализованные подписи (рек. §8 #3, #5).
- **Усечение больших:** `organizations` (199), `individuals` (96),
  `industryDirections` (111), `commissions` (30), `interest-rates` (29),
  `creditTerms` (20), `loan-termses` (19) → 6–8 представительных строк +
  пометка `«показано N из M»` (no silent truncation).
- **Пустые справочники (13)** — секция рендерится с пустым гридом и плашкой
  `«Справочник не наполнен»` (отражает находку §7 без as-is-маркеров: это нейтр.
  empty-state, а не дефект-бейдж).

## 6. Интерактив (минимальный, переиспользуемый движок)

Один обобщённый JS-движок на все секции (DRY — это то, что копируют другие мокапы):

- **Фильтр в sidebar** — живой поиск по названиям справочников.
- **`+ Добавить условие поиска`** — раскрывает строку фильтра грида (поле →
  фильтрует строки секции по подстроке). Демонстрирует паттерн, без конструктора.
- **`Просмотр` / двойной клик по строке** — открывает **общую read-only модалку**
  с полями строки (рек. §8 #1 «Просмотр» для всех).
- **`Изменить`** — та же модалка, поля редактируемы (визуально; без бэкенда).
- **`Создать`** — пустая модалка по колонкам секции.
- **`Удалить`** — подтверждение, строка убирается из DOM (клиентски).
- **Picker-демо** — в группе picker-only кнопка «Открыть как picker» показывает
  полноэкранный диалог **в режиме только выбора** (рек. §8 #2 / P2-R9): радио/клик
  выбирает значение, без Создать/Изменить/Удалить. Это эталонный picker-компонент
  для других мокапов.

Глубину держим минимальной: движок читает определения колонок секции и строит
модалку/фильтр обобщённо. Никакой персистентности.

## 7. Группировка секций (порядок)

1. **Решения и получатели** — credit-order-states/types, order-document-types,
   entity-document-*, document-package-*, applied-entity-*
2. **Источники, валюты, кредит** — order-term-funds, order-term-currencies,
   loan-types, industryDirections, work-sectors, loan-credit-lines,
   loan-type-repayment-accounts, loan-grace-periods
3. **Ставки** — interest-rates, loan-percent-rates, order-term-floating-rate-types
4. **Платежи и расчёты** — order-term-frequency-types, order-term-rate-periods,
   order-term-days-methods, order-term-accr-methods, order-term-transaction-orders
5. **Договоры и залог** — agreement-templates, agreement-template-codes,
   collateralTypes
6. **Субъекты и заёмщик** — employees, commissions, individuals, organizations,
   borrower-groups, debtor-types, organization-forms, loan-payment-capacity-groups
7. **Кредит и график** — loan-states, loan-redemption-accounts, loan-termses,
   creditTerms, calculationCoefficients, installment-states, payment-types
8. **Товары и счета** — good-types, destination-accounts, payment-purpose-requisites
9. **Picker-only (нет пункта меню)** — Цели кредита, Уровни проверки документов,
   Типы расчёта процентов

## 8. Переиспользование другими мокапами

Что копируется из этого файла:
- блок CSS-токенов `--asubk-*` (один источник);
- грид-разметку и классы бейджей `.badge.success/warning/info/error`;
- **picker-движок** (режим только выбора) — для lookup-полей форм;
- **готовые таблицы значений** каждого справочника — вставляются в picker-диалоги
  и select-поля сущностей (Госрешение, Программа, Заявка, Заёмщик, Кредит…).

## 9. Вне рамок (YAGNI)

- Реальный CRUD/персистентность, серверная валидация, пагинация-движок.
- Конструктор фильтра jmix (только демо-строка поиска).
- Справочники внешних модулей (субсидии, поручительства, гео, безопасность) — §1
  ТЗ их выносит за границы.
- as-is дефект-маркеры на секциях (выбран чистый to-be).

## 10. Критерий готовности

- Файл открывается, sidebar-якоря скроллят к 50 секциям, активный пункт следует
  за скроллом, фильтр прячет несовпадающие.
- Все 47+3 справочника присутствуют, у каждого тулбар+грид, данные перенесены и
  вычищены, статусы — бейджами, большие усечены с пометкой, пустые — с плашкой.
- Просмотр/Изменить/Создать/Удалить и picker-демо работают клиентски.
- Визуально совпадает с дизайн-системой loan-program/decisions.
