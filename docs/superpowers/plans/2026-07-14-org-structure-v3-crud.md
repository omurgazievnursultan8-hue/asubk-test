# План: оргструктура v3 — довести модуль до полного цикла операций

Мокап: `mockups/org-structure/org-structure-v2.html`
Аудит, откуда взяты пункты: `scripts/inspect/org-structure-v2-audit.mjs`
Регресс: `scripts/inspect/org-structure-v2-check.mjs`

Проблема одной строкой: **реализовано только создание**. Изменить нельзя ничего,
закрыть можно только пустой лист, а девять валидаций пропускают заведомо неверные данные.

---

## Э1. Модель: штатное расписание становится историчным

`POSITIONS` теряет `units`/`isHead` и получает жизненный цикл; атрибуты выносятся в версии —
тот же приём, что уже применён к названию (`NAME_VER`) и подчинённости (`PARENT_VER`).

```js
POSITIONS = [{ id, unitId, titleId, from, to }]          // тождество + срок жизни единицы
POS_VER   = [{ posId, units, isHead, from, to }]         // версии числа ставок и признака руководителя
```

Новые резолверы: `posActive(p,d)`, `posAttrAt(posId,d)`, `unitsAt(posId,d)`, `isHeadAt(posId,d)`,
`unitPositionsAt(unitId,d)`. `headPosOf(unitId)` → `headPosOf(unitId, d)` — правится каждый вызов
(`head`, `chainUp`, `metricsAt`, `problemsAt`, `renderTree`, `miniTreeHtml`, `submitAddPos`).

Проверка: единица, заведённая 2026-07-11, на 2021-01-01 не видна.

## Э2. Общая форма (`openForm`) вместо десяти новых модалок

Одна модалка `#gfBack` + декларативный конфиг (`fields` → text/date/number/select/checkbox/checklist,
`onSubmit(v)` возвращает строку-ошибку или `null`). На ней строятся все новые действия ниже.

## Э3. Историчность становится действием, а не демо-данными

* **Переименование** — закрывает текущую версию `NAME_VER`, открывает новую с выбранной даты.
* **Переподчинение** — то же с `PARENT_VER` (при первом изменении базовое подчинение
  материализуется в версию). Запреты: цикл, сам себе, верхний уровень, закрытое подразделение.
* **Территория** — не свободный текст, а справочник `TERRITORIES` (7 областей + 2 города),
  выбор чекбоксами. Убирает расхождение «Джалал-Абадская обл.» ≠ «Джалал-Абадская область».

## Э4. Кадровый оборот в обе стороны

* **Освободить** (`ASSIGN.to`), **прекратить и.о.**, **продлить и.о.** (`ACTING.to`).
* **Изменить штатную единицу** — новая версия `POS_VER` (ставки, признак руководителя).
* **Закрыть штатную единицу** (`POSITIONS.to`) — блокируется действующими назначениями.
* Три мёртвые кнопки в «Требует внимания» получают обработчики:
  «Назначить» → форма назначения на руководящую единицу, «Продлить» → продление и.о.,
  «Добавить штатную единицу» → форма новой единицы в нужном подразделении.

## Э5. Закрытие подразделения доводится до конца

Форма закрытия: дата + **кому переданы функции** (выбор из действующих) + флажок
«передать территорию правопреемнику» + подтверждение. Новая проверка №4: **события после даты
закрытия** (переименование/переподчинение/назначение позже даты) — блокировка.
Появляется **отмена закрытия**. Событие наружу сохраняет контракт `unit.liquidated(id, дата, правопреемник?)`,
добавляются `unit.renamed` / `unit.moved` / `unit.reopened`.

## Э6. Валидации, которые сейчас пропускают неверные данные

| Правило | Как сейчас | Как станет |
|---|---|---|
| Сумма ставок на единице | не проверяется (200 %) | сумма занятых + новая ≤ числу ставок, на каждой границе интервала |
| Одно основное место | проверка в точке `from` | пересечение интервалов (`overlaps`) |
| Совместительство на ту же единицу | дубль проходит | пересечение по (сотрудник × единица) любого типа |
| Даты назначения | любые | внутри срока жизни подразделения и штатной единицы |
| Дата закрытия | любая | не раньше событий и не раньше создания |
| Второй верхний уровень | создаётся молча | запрещён, если корень на дату уже есть |
| Дробные ставки | 2,5 проходит | целое ≥ 1 |
| Срок и.о. | бесконечный | не больше `MAX_ACTING_DAYS = 183` |
| Дубль названия | проходит | запрещён среди действующих |
| Закрытие подразделения | один клик | форма с подтверждением |

## Э7. Расхождение метрик вакансий

Сейчас «Вакансии всего» считает единицы без назначений, а «в т.ч. руководителя» —
единицы без основного **и без и.о.**, поэтому и.о. беззвучно убирает подразделение из рабочего списка.
Правило станет одно: **и.о. вакансию не закрывает, он её прикрывает**. Плитка показывает
незанятые *ставки*; список «Вакансия руководителя» подразделение не теряет, а получает колонку
«Прикрыто и.о.».

## Э8. Справочники, сотрудники, поиск, экспорт

* Должность: изменить (название, категория, признак) и удалить неиспользуемую.
* Сотрудник: завести (заглушка чужого модуля — иначе принять нового человека нечем).
* Поиск: по структуре, по должностям, по сотрудникам.
* Экспорт CSV: структура на дату и штатное расписание подразделения.
* Печать: на карточке подразделения (на «Обзоре» кнопка переименовывается в «Печать обзора»).
