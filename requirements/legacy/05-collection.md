# P5. Взыскание (legacy) — процедура, фазы, события, претензии

> Снято 2026-07-12, браузер `claude-in-chrome`, хост `85.113.29.29:8080`,
> пользователь Сламкулов А.О. Вендор: **UlutSoft**.
> Пример — реальная процедура debtor-37838 / procedure-64940 (Взыскание задолженности,
> цепочка Претензия → Иск → Заявление в суд, loan-46883, 159 500,00 KGS).
> Связь: взыскание видно в карточке заёмщика (вкладка «Принудительное взыскание»,
> `01-debtor.md` §2 вкл.4) и в карточке кредита (вкладка ФАЗЫ, `02-loans.md` Tab8).

**Модель взыскания:** заёмщик → **Процедура взыскания** (`collectionprocedure`) →
**Фазы** (`collectionphase`, цепочка по видам) → **События** (`collectionevent`,
судебные заседания/извещения внутри фазы). Плюс два «среза» списка фаз (судебный и
ПССИ/исполнители) и отдельный срез претензий.

---

## 1. Список фаз взыскания — `/collectionPhaseViews`

Меню: **Взыскание → Список стадий взыскания**. Заголовок «Фазы взыскания». **100 записей.**
Фильтры: Наименование заемщика · **Вид фазы** · **Гр. платежесп.** · Область · Район · ▾.
Колонки: Просмотр(→ карточка ПРОЦЕДУРЫ) · Наименование заемщика(→) · Начал. дата ·
Начал. сумма · Конеч. сумма · **Вид** · **Статус** · Конеч. дата.
Вид: **Претензия**, **Заявление в суд** (и др. судебные). Статус: На исполнении,
Опр. суда (Без рассмот.). Пример: debtor-140 · Претензия · 2 805 178,00 · На исполнении.

⚠️ «Просмотр» открывает не фазу, а **процедуру** (`/manage/debtor/{d}/collectionprocedure/{id}/view`).

## 2. Список стадий взыскания (2) — `/collectionPhaseViews/second`

Меню: **Взыскание → Список стадий взыскания(2)**. Заголовок «Фазы взыскания».
Крошки: `List of CollectionPhaseViews` ⚠️ (не переведено). **1000 записей.**
Фильтры: Наименование заемщика · Гр. платежесп. · Вид фазы · Область · Район · ▾.
Колонки: **#** (id-ссылка) · Наименование заемщика · Начал. дата · Начал. сумма · Вид · Статус.
Вид здесь = **«Письмо в ПССИ»** (стадия судебных исполнителей / исполнительное
производство — отдельно от судебной цепочки списка №1).
Статус: Возвр. отсут. зал. имущества долж. · На исполнении · Возвр. принятие особ.
решения · Возвр. отсут. имущ. должника · Возврат (Отс. ден. средств).

⚠️ **Дефекты:** пагинация и навигация на **кыргызском** («Жалпы 1 000 саптын ичинен…»,
«Мур.», «Кийин.») на русской странице · даты вида **`20.08.0001`** (битый год) · числа
без разделителей разрядов (`118051`).

## 3. Карточка процедуры взыскания — `/manage/debtor/{d}/collectionprocedure/{id}/view`

Пример: `…/37838/collectionprocedure/64940/view`. Заголовок H1 = «Взыскание
задолженности, В работе отр. отд., 10.05.2024». Крошки: `Заемщики › debtor-37838 ›
Процедура взыскания`.

Содержит **таблицу фаз** (цепочку): поиск «Искать» + кнопка **«+Добавить новую фазу
взыскания»** (`…/collectionphase/0/save`).
Колонки: Просмотр(→ карточка фазы) · **Вид** · Дата фазы · Сумма фазы · Статус фазы ·
Дата завершения · Сумма завершения · [иконка **«Изменить отделение»**
`…/collectionphase/{id}/department/change`].

Пример цепочки (снизу вверх по времени):
| Вид | Дата | Сумма | Статус | Дата зав. | Сумма зав. |
|-----|------|-------|--------|-----------|-----------|
| Претензия | 10.05.2024 | 153 700,00 | На исполнении | — | 0,00 |
| Иск | 04.10.2024 | 159 500,00 | Полное погашение | 09.12.2024 | 159 500,00 |
| Заявление в суд | 09.12.2024 | 159 500,00 | Опр. суда (Без рассмот.) | 09.12.2024 | 159 500,00 |

**Порядок цепочки:** Претензия → Иск → Заявление в суд → (далее Письмо в ПССИ /
исполнительное производство).

## 4. Карточка фазы взыскания — `/manage/debtor/{d}/collectionprocedure/{proc}/collectionphase/{id}/view`

Пример: `…/collectionphase/82647/view`. Заголовок H1 = «Иск, Полное погашение, 04.10.2024».
Кнопки: **Редактировать**, **Изменить статус** (teal), **Отменить фазу**. **5 вкладок.**

### Вкладка 1 — «Общ. информация» (3 колонки)
| Колонка 1 | Колонка 2 | Колонка 3 |
|-----------|-----------|-----------|
| Дата инициирования = 04.10.2024 | Вид фазы = Иск | `LABEL.COLLECTION.PHASE.CLOSE.AMOUNT` ⚠️ = 159 500,00 |
| Всего погашено = 160 000,00 | Статус фазы = Полное погашение | Номер документа _(пусто)_ |
| На сумму = 159 500,00 | | Внесено = Танаев Урматбек Шарабидинович (24.10.2024) |
| Номер документа _(пусто)_ | | Изменён = Тукинова Айгуль Сагынбековна (22.01.2025) |

⚠️ «Номер документа» дублируется (колонки 1 и 3).

### Вкладка 2 — «Фаза взыскания» (широкая фин. разбивка по кредиту)
Колонки: `reg number` ⚠️(англ) · Основная сумма · % · Штрафы · Итого · Погашено осн.
сумма · Погашено % · Погашено штрф. · Погашено · Завершено осн. сумма · Завершено % ·
Завершено штрф. · Завершено.
Пример: loan-46883 · 159 500 · 0 · 0 · 159 500 · 160 000 · 0 · 0 · 160 000 · 159 500 · 0 · 0 · 159 500.

### Вкладка 3 — «Кредит»
Привязанный кредит(ы) фазы.

### Вкладка 4 — «Доп. информация» = **СОБЫТИЯ фазы** (`collectionevent`)
Кнопка **«+Добавить доп.информацию»**. Колонки: Просмотр(→) · Дата · # · Статус · Вид ·
Куратор · Место · Время + [⋯ меню].
Пример: 24.10.2024 · на рассмотрении · **Судебный процесс** · Тукинова Айгуль ·
**Ноокат райондук соту** · 05.11.2024 10:30.

### Вкладка 5 — «Информация»
Вложения (файлы).

## 5. Список событий по фазам — `/collectionEventViews`

Меню: **Взыскание → Список событий**. Заголовок «События по фазам».
Фильтры: Наименование заемщика · **Вид события** · **Ф.И.О куратора** · **С даты** ·
**по дату** · ▾. (единственный список P5 с диапазоном дат)
Колонки: Просмотр(→) · Наименование(→) · **Фаза** (напр. «Иск от 10.01.2025 · 20 890,00») ·
**Событие** (Судебный процесс от {дата} / Извещение о назначении судебного процесса от {дата}) ·
**Куратор** (ФИО + «участвует» + доп. ФИО участников) · **Место** (Жети-Огузский рай.суд) ·
**Время** (дата - время заседания) · **Статус** (на рассмотрении + подстатус: «Назначение
для рассмотр.-ия по существу», «Отложено в связи с неявкой ответчика» и т.п.).

Это трекинг судебных заседаний: место (районный суд), дата/время, участники, исход.

## 6. Список претензий — `/collectionPhaseViews/claim`

Меню: **Документы → Список претензий**. Заголовок «Фазы взыскания». **100 записей.**
Фильтры: Наименование заемщика · Гр. платежесп. · Область · Район · ▾.
Колонки: **#** (чип статуса, красный **«Not registered»** ⚠️ англ.) · Ном. документа ·
Дата рег. · Наименование заемщика(→) · Начал. сумма · Нач. дата · Конеч. сумма ·
[🗑 удалить + ✏️ править — inline] · Конеч. дата.
Все примеры «Not registered» — незарегистрированные претензии (черновики): Ном.
документа и Дата рег. пусты, доступны inline удаление/правка.

## Итоги P5 (что важно НЕ потерять в новой системе)

1. **Иерархия:** заёмщик → процедура взыскания → фазы (цепочка) → события (заседания).
2. **Цепочка фаз по видам:** Претензия → Иск → Заявление в суд → Письмо в ПССИ
   (исполнительное производство). Два «среза» списков — судебный (№1) и ПССИ (№2).
3. **Статусы фаз** — На исполнении / Полное погашение / Опр. суда (Без рассмот.) /
   Возврат (отс. имущества/ден. средств) и др.; кнопки «Изменить статус», «Отменить фазу».
4. **Финансовая разбивка фазы** — Основная/%/Штрафы × (начислено / погашено / завершено).
5. **События (заседания)** — вид, место (районный суд), дата/время, куратор + участники,
   статус с подстатусом (исход заседания). Фильтр по диапазону дат.
6. **Смена отделения** фазы (`department/change`) — реорганизация ведения.
7. **Претензии** — отдельный срез с регистрацией (registered/not registered) и inline
   удалением/правкой черновиков.
8. **Реальные ФИО кураторов** в фазах/событиях (Танаев, Тукинова, Эрнисов…) — в отличие
   от owner-XXXXX в залоге.

**Дефекты-кандидаты (→ `notes/qa-findings.md`):** непереведённые ключи
`LABEL.COLLECTION.PHASE.CLOSE.AMOUNT`, `reg number`, крошка `List of CollectionPhaseViews`,
чип `Not registered`; **кыргызская пагинация** на RU-странице (`/second`); битые даты
`20.08.0001`; числа без разделителей разрядов (`/second`); дубль метки «Номер документа»
на карточке фазы.
