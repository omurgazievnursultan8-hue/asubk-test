## 1. Список погашений — `/paymentView/list`

Пункт меню: **Кредитование → Список погашений**. Крошки: `🏠 › Платежи`.
Заголовок вкладки браузера — «Просмотр».

**Объём:** «Отображение 1–10 из **100** записей».

### Панель фильтров (зелёная)
| Контрол | Тип | Значения |
|---------|-----|----------|
| Наименование | текст | placeholder «Наименование» (ФИО/название заёмщика) |
| Номер платежа | текст | placeholder «Номер платежа» |
| Область | выпадающий | Все + 9 областей (Баткенская, г.Бишкек, Джалал-Абадская, Иссык-Кульская, Нарынская, Ошская, Прочие, Таласская, Чуйская) |
| Район | выпадающий | Все + полный справочник районов/городов КР |
| ▾ (правый край) | сворачиваемая панель | доп. фильтры (скрыты за стрелкой) |

### Таблица
| Колонка | Содержимое |
|---------|-----------|
| Просмотр | иконка-стрелка → **карточка КРЕДИТА** `/manage/debtor/{d}/loan/{l}/view` (НЕ карточка платежа!) |
| Наименование | ФИО/название → `/manage/debtor/{d}/view` |
| Номер | номер платёжного документа (напр. `1057`, `235100177`, `б/н`) |
| Сумма | **сумма ПО ДОГОВОРУ кредита** (одинакова для всех платежей одного кредита — напр. 4 785 302,74 у всех строк debtor-140) |
| Всего | **сумма самого платежа** (напр. 589 460,00 / 7 000,00) |
| Дата | дата платежа |

**Пагинация:** первая/пред/1-6/…/след/последняя. Размер страницы 10/20/30/50/100.

**Наблюдение (для сравнения):**
- «Просмотр» ведёт не на платёж, а на **кредит** — отдельной точки входа в карточку
  платежа из этого списка нет (только через кредит → вкладка ПЛАТЕЖИ → стрелка).
- Колонка **«Сумма»** показывает сумму договора, а не платежа — заголовок вводит в
  заблуждение (сам платёж в колонке «Всего»). ⚠️ кандидат в дефекты (UX).

## 2. Список графиков — `/paymentScheduleView`

Пункт меню: **Кредитование → Список графиков**. Крошки: `🏠 › Графики`.

**Пустой по умолчанию** — таблица не отображается, пока не задан фильтр (ленивая
загрузка). Видна только панель фильтров.

### Панель фильтров
| Контрол | Тип |
|---------|-----|
| Наименование заемщика | текст |
| Область | выпадающий (9 областей) |
| Район | выпадающий (справочник районов) |

вкладка **ГРАФИК** (`02-loans.md` §2, Tab4): №/#/Дата/Освоение/Осн.сумма/Проценты/
Нак.проценты/Нак.штраф/Статус + строка итогов; кнопки экспорта PDF/Excel
(`label.getPdf`/`label.getExcel` ⚠️) и **+Добавить график**.

## 4. Карточка платежа — `/manage/debtor/{d}/loan/{l}/payment/{payId}/view`

Пример: `/manage/debtor/140/loan/484/payment/53166109/view`.
Заголовок H1 = номер платежа («1»). Заголовок вкладки браузера — «Погашение».

⚠️ **Дефект:** крошки и метка страницы показывают `Заемщик › Список заемщиков ›
Заемщик` — неверный контекст (это платёж, не заёмщик). Кандидат в `qa-findings.md`.

**3 вкладки:**

### Вкладка 1 — «Общая информация»
3 колонки:
| Поле | Значение (пример) |
|------|-------------------|
| `LABEL.PAYMENT.DATE` ⚠️ | 03.08.2011 |
| Итого | 27000.0 |
| Осн.сумма | 27000.0 |
| Проценты | 0.0 |
| Штрафы | 0.0 |
| Вид платежа | денежными средствами |
| Внесено | - |
| Изменён | - |

⚠️ Числа сырые `.0` (27000.0), а не форматированные как в списке (27 000,00) — косметика.

### Вкладка 2 — «Информация»
Вложения к платежу. Поиск «Искать…», кнопка **«Добавить информацию»**.
Колонки: Название файла / Название. Пусто («Записей не найдено»).

### Вкладка 3 — `LABEL.HISTORY` ⚠️
Журнал изменений платежа (аудит). Заголовок вкладки не переведён.

## Итоги P3 (что важно НЕ потерять в новой системе)

1. **Два глобальных списка** — Погашения (`/paymentView/list`, 100 записей) и Графики
   (`/paymentScheduleView`, ленивый). Оба с гео-фильтрами Область/Район.
2. **Погашения не имеют своей точки входа** — список ведёт на кредит; карточка платежа
   доступна только через кредит → ПЛАТЕЖИ → стрелка.
3. **Разложение платежа** на Осн.сумма / Проценты / Штрафы — ключевая структура.
4. **Вид платежа** (денежными средствами / …) — классификатор, вынести в новую систему.
5. **Вложения к платежу** (вкладка Информация) + **журнал изменений** (история).
6. **Итоги по кредиту** внизу вкладки платежей: СУММА / ОСН.СУММА / ШТРАФ / ПРОЦЕНТ.
7. Платежи **read-only** из кредита — ввод/регистрация платежа происходит в другом
   потоке (расчёт/погашение) — проследить в P5.

**Дефекты-кандидаты (→ `notes/qa-findings.md`):** непереведённые ключи
`LABEL.PAYMENT.DATE`, `LABEL.HISTORY`; неверные крошки/заголовок «Заемщик» на карточке
платежа; колонка «Сумма» в списке погашений показывает сумму договора, а не платежа;
сырой формат чисел `27000.0` на карточке платежа.
