# P4. Залог (legacy) — предметы, договор, обследование, снятие ареста, поручители

> Снято 2026-07-12, браузер `claude-in-chrome`, хост `85.113.29.29:8080`,
> пользователь Сламкулов А.О. Вендор: **UlutSoft**.
> Пример — реальный договор залога debtor-54668 / ГН-31-2024-2951998 / agreement-38710 /
> предмет-52100 (земельный участок, зал.стоимость 450 000,00 KGS, связан с loan-60518).
> Связь: залог виден и в карточке заёмщика (вкладка «Залоговое обеспечение», `01-debtor.md` §2)
> и в карточке кредита (вкладка ЗАЛОГ, `02-loans.md` Tab6).

**Иерархия залога:** заёмщик → **договор залога** (`collateralagreement`) → **предмет
залога** (`collateralitem`). Договор связывает залог с одним или несколькими **кредитами**.

## 2. Карточка предмета залога — `/manage/debtor/{d}/collateralagreement/{agr}/collateralitem/{item}/view`

Пример: `…/54668/collateralagreement/38710/collateralitem/52100/view`.
Заголовок H1 = наимен. залога («земельный участок»). Крошки: `Заемщики › debtor-54668 ›
ГН-31-2024-2951998 › Предмет залога`. Кнопка **Редактировать**. **4 вкладки.**

### Вкладка 1 — «Общая информация» (3 колонки)
| Колонка 1 | Колонка 2 | Колонка 3 |
|-----------|-----------|-----------|
| `LABEL.COLLATERALITEM.INSPECTIONSTATUS` ⚠️ = «1. Подлежит обследованию» | Залогодатель = owner-58534 | Вид недвижимости = земельный участок |
| Вид залога = Участок | Правоуст. документ = «Госакт о праве частной собст. от 20.01.2023г. №1713» | Идентификационный код = 2-01-17-1001-3556 |
| Кол-во = 1.0 шт. | Состояние = [кнопка] «1. Обследование (раз в 3 года)» | Общая площадь (кв.м.) |
| Примечание | Место наложения ареста = owner-28986 | Жилая площадь (кв.м.) |
| Недоработка по оформлению | Внесено = Эмильева Упол (22.01.2025) | Площадь застройки (кв.м.) |
| Залоговая стоимость = 450 000,00 | Изменён = - | Зем.участок мерою (кв.м.) = 500.0 |
| Оценочная стоимость = 450 000,00 | | Адрес = с.Теплоключенка, ул.Пректируемая 7 дом 1 |
| | | Дата постройки |

**Кнопка «Состояние»** (зелёная) — смена периодичности/статуса обследования.
Поля площадей зависят от вида недвижимости (для участка заполнено «мерою», а не «общая»).

### Вкладка 2 — «Акты обследования»
Поиск «Искать» + кнопка **«+Добавить акт обследования»**.
Колонки: `#` / Дата обследования / Примечание / Результат. (у примера пусто)

### Вкладка 3 — «Снятие с ареста» (заголовок «Снятие залога»)
Поиск + 2 кнопки: **«+Добавить снятие с ареста»** (синяя) и **«Снятие с ареста (2)»** (красная — второй тип/канал снятия).
Колонки: `#` / Дата обследования / Документ / На основании / Примечание / Снято.

### Вкладка 4 — «Информация»
Вложения. Поиск + **«Добавить информацию»**. Колонки: Название файла / Название.

## Итоги P4 (что важно НЕ потерять в новой системе)

1. **Трёхуровневая модель:** заёмщик → договор залога → предмет залога; договор ↔ кредиты (M:N).
2. **Две стоимости предмета** — залоговая и оценочная (раздельно).
3. **Тип-зависимые поля недвижимости** — площади (общая/жилая/застройки/мерою), идент. код, адрес, дата постройки, правоуст. документ.
4. **Цикл обследования** — статус обследования + периодичность («раз в 3 года») + акты с результатом (Удовл./Хорошее/Неудовл.).
5. **Арест и снятие** — номер налож. запрета, место ареста, нотариус; снятие ареста двумя каналами («Снятие (2)») + `label.agreement.arrest.free` на уровне договора.
6. **Доп. соглашения** и **пересмотр договора** (agreementReview) — отдельные вкладки.
7. **Поручительство** — параллельный залогу вид обеспечения (заёмщик · нотариус · поручитель), 730 договоров.
8. **Вложения** (Информация) на каждом уровне.
9. **Печатная форма** «Извещение (утрата)» из карточки заёмщика.

**Дефекты-кандидаты (→ `notes/qa-findings.md`):** непереведённые ключи —
`LABEL.COLLATERALITEM.INSPECTIONSTATUS`, `LABEL.ID`, `label.agreement.arrest.free`,
`label.add.collateral.additionalAgreement`, `LABEL.COLLATERALAGREEMENT.TAB.AGREEMENTREVIEW`,
`label.guarantors`, `label.guarantor.name`; сырые числа `1.0 шт.` / `500.0`; owner-XXXXX
(нотариус/залогодатель/поручитель) — placeholder-имена вместо реальных ФИО.
