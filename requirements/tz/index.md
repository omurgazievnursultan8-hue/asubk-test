# ТЗ — Подсистема «Система кредитования» (as-is)

> Функциональная спецификация as-is. Стенд: https://fkftest.okmot.kg/ (admin/admin).
> Все факты верифицированы live-инспекцией, см. `scripts/inspect/tz/`.

## Назначение и границы

**Подсистема «Система кредитования»** автоматизирует полный жизненный цикл государственного
кредитования в АСУБК — от нормативного основания до обслуживания выданного займа.

### Что входит в подсистему (7 этапов)

| Этап | Сущность | Маршрут |
|---|---|---|
| 01 | Госрешение | `/gov-decisions` |
| 02 | Кредитная программа | `/loan-programs` |
| 03 | Заявка / Комиссия | `/loan-applications`, `/loan-application-commissions` |
| 04 | Заёмщик | `/loan-applicants` |
| 05 | Кредит | `/loansCredit`, `/loan-credits/{id}` |
| 06 | Транши / Освоение | `/sub-loans`, `/disbursements` |
| 07 | Обслуживание (Платежи, Резервы, Реестр) | `/payments`, `/loan-reserves`, `/loan-ledgers` |

### Что НЕ входит — внешние модули (ссылки только)

Следующие модули приложения связаны с кредитованием, но документируются отдельно
и не являются частью настоящей спецификации:

- **Поручительства** — поручительства и сюрети по кредитам (`requirements/overview.md`)
- **Корпоративное управление** — реестр компаний, KPI, отчётность (`requirements/overview.md`)
- **СУГС** — субсидии и компенсационные выплаты (`requirements/overview.md`)
- **Взыскание задолженности** — досудебное, судебное и исполнительное производство (`requirements/overview.md`)

## Архитектура и доступ

### Технологический стек

Приложение АСУБК реализовано на платформе **Jmix поверх Vaadin** (Java SPA).
Пользовательский интерфейс — русскоязычный бэк-офис для внутренних сотрудников Фонда;
адреса вида `/gov-decisions`, `/loan-programs` и т. д. — это маршруты Vaadin Views,
открывающиеся без перезагрузки страницы. Подтверждено инспекцией 2026-06-17 (`requirements/overview.md`).

### Модель доступа

Доступ управляется **штатной безопасностью Jmix** (см. `guides/access-control.md`):

| Уровень | Назначение |
|---|---|
| **Ресурсные роли** | Контролируют пункты меню, экраны, кнопки CRUD, поля сущностей |
| **Роли уровня строк** | Ограничивают, какие именно записи видит пользователь (JPQL-фильтр) |

Роли назначаются пользователям через **Безопасность → Ресурсные роли**
и **Безопасность → Роли уровня строк** (источник: `guides/access-control.md`).

На тест-стенде зафиксированы следующие роли, относящиеся к кредитному блоку:

| Роль (название) | Код | Источник | Состояние |
|---|---|---|---|
| `Специалист кредитного отдела` | не зафиксирован | База данных | Роль без политик — не даёт доступа (проверено 2026-06-20, `guides/access-control.md`) |
| `СУГС` | не зафиксирован | База данных | Роль без политик — не даёт доступа (проверено 2026-06-20, `guides/access-control.md`) |
| `банки` | не зафиксирован | База данных | Полный CRUD к сущности `Bank` — эталонный пример правильно настроенной роли (проверено 2026-06-20, `guides/access-control.md`) |
| `Full Access` | — | Аннотированный класс (Java-код) | Полный доступ; только для чтения в UI |
| `ui-minimal` | — | Аннотированный класс | Минимальный UI-доступ; только для чтения в UI |

## Модель данных

Цепочка сущностей кредитного цикла с кардинальностями, синтезированная из раздела 7 «Связи»
каждого этапа:

```
Госрешение ─1:N→ Кредитная программа ─1:N→ Заявка ─N:1→ Заёмщик
                                              │
                                              │ (при одобрении)
                                              ▼
                                        Заявка(одобрена) ─1:1→ Кредит ─1:N→ Транш ─1:N→ Освоение
                                                                  │
                                                    ┌─────────────┼─────────────┐
                                                    ▼             ▼             ▼
                                                 Платёж        Резерв    Запись реестра
                                                                           (LoanLedger)
```

**Источники по каждой стрелке:**

| Связь | Кратность | Источник |
|---|---|---|
| Госрешение ─1:N→ Кредитная программа | 1:N | [01](01-gosreshenie.md) §7; [02](02-kreditnaya-programma.md) §7: «каждая программа ссылается ровно на одно госрешение» |
| Кредитная программа ─1:N→ Заявка | 1:N | [02](02-kreditnaya-programma.md) §7; [03](03-zayavka-komissiya.md) §7: «каждая заявка создаётся в рамках одной программы» |
| Заявка ─N:1→ Заёмщик | N:1 | [03](03-zayavka-komissiya.md) §7; [04](04-zaemshchik.md) §7: «один заёмщик может иметь несколько заявок» |
| Заявка(одобрена) ─1:1→ Кредит | 1:1 (0..1) | [03](03-zayavka-komissiya.md) §7; [05](05-kredit.md) §7: кредит порождается из одобренной заявки; кнопки «Создать» в списке кредитов нет |
| Кредит ─1:N→ Транш | 1:N | [05](05-kredit.md) §7; [06](06-transhi-osvoenie.md) §7: вкладка «Транши» карточки кредита |
| Транш ─1:N→ Освоение | 1:N | [06](06-transhi-osvoenie.md) §7: «поле `Disbursement.subLoan` в карточке освоения ссылается на родительский транш» |
| Кредит ─1:N→ Платёж | 1:N | [05](05-kredit.md) §7; [07](07-obsluzhivanie.md) §7: поле «Кредит» карточки платежа |
| Кредит ─1:N→ Резерв | 1:N | [05](05-kredit.md) §7; [07](07-obsluzhivanie.md) §7: колонка `Loan` списка резервов |
| Кредит ─1:N→ Запись реестра (LoanLedger) | 1:N | [05](05-kredit.md) §7; [07](07-obsluzhivanie.md) §7: колонка «Кредит» реестра |

Дополнительная связь: **Заявка ─1:N→ Комиссия по заявкам** (одна заявка может иметь несколько
записей комиссии — кредитная и залоговая работают параллельно, источник: [03](03-zayavka-komissiya.md) §7).

## Глоссарий

| Термин | Определение |
|---|---|
| **Госрешение** | Корневая сущность кредитной подсистемы. Нормативно-правовое основание (постановление, распоряжение и т. д.), разрешающее государственное кредитование; фиксирует номер и дату акта, вид решения и статус согласования. Источник: [01](01-gosreshenie.md) §1. |
| **Кредитная программа** | Шаблон кредитного продукта, определяющий параметры (сумма, срок, ставки, штрафы, льготные периоды, документы, залог), на основе которых создаются заявки и кредиты. Ссылается ровно на одно госрешение. Источник: [02](02-kreditnaya-programma.md) §1. |
| **Заявка** | Запрос субъекта на кредит в рамках конкретной кредитной программы; содержит запрашиваемую сумму и срок, метод погашения, льготные периоды. Проходит цикл: Новый → На рассмотрении → Одобрено / Отклонено. Источник: [03](03-zayavka-komissiya.md) §1. |
| **Комиссия** | Запись голосования коллегиального органа по заявке (кредитная или залоговая). Содержит состав (члены + председатель), индивидуальные голоса и финальное решение председателя. Источник: [03](03-zayavka-komissiya.md) §1. |
| **Заёмщик** | Субъект (физическое или юридическое лицо), привязанный к одной или нескольким заявкам и кредитам. Хранит ИНН, тип лица, отрасль, контакты, адрес и счётчик активных кредитов. Источник: [04](04-zaemshchik.md) §1. |
| **Кредит** | Центральная операционная сущность; создаётся автоматически из одобренной заявки. Карточка содержит 11 вкладок: договор, условия, график, транши, резерв, платежи, детальный расчёт. Источник: [05](05-kredit.md) §1. |
| **Транш** | Плановая доля кредита (`SubLoan`), предназначенная конкретному заёмщику. Суммарный объём траншей не превышает одобренной суммы кредита; отслеживает распределённый и доступный остаток. Источник: [06](06-transhi-osvoenie.md) §1. |
| **Освоение** | Фактическая выборка денежных средств в рамках транша (`Disbursement`). Содержит сумму, срок, ставку, периодичность и генерирует собственный аннуитетный или иной график платежей. Источник: [06](06-transhi-osvoenie.md) §1. |
| **Резерв** | Ежемесячно начисляемая провизия (резерв на возможные потери) на неосвоенную сумму кредита по ставке «Процент за резерв»; формула: `Неосвоенная сумма × Ставка / 100`. Источник: [07](07-obsluzhivanie.md) §1, §5.2. |
| **LoanLedger** | Неизменяемый реестр финансовых событий кредита (`/loan-ledgers`): начисления основного долга, процентов, пени, платежи и списания. Каждая запись фиксирует тип события, дату и суммы по составляющим. Источник: [07](07-obsluzhivanie.md) §1, §6.3. |
| **Аннуитет** | Метод погашения кредита равными периодическими платежами; PMT = PV × r / (1 − (1 + r)^(−n)). Применяется на уровне кредита (вкладка «График погашений») и на уровне освоения (карточка `Disbursement`). Источник: [05](05-kredit.md) §5.2; [06](06-transhi-osvoenie.md) §5.3. |
| **Льготный период** | Отсрочка платежей по одному или нескольким компонентам: основной долг, начисление процентов, уплата процентов. Задаётся тремя независимыми флагами в программе (вкладка 5) и в заявке; числовое значение длительности хранится в условиях транша (`Льгота по осн. долгу, мес.` и др.). Источник: [02](02-kreditnaya-programma.md) §5.6; [03](03-zayavka-komissiya.md) §3.1; [06](06-transhi-osvoenie.md) §3.3. |
