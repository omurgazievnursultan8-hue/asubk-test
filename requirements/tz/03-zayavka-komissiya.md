## 1. Назначение

**Заявка (Кредитная заявка)** — третья сущность кредитного цикла. Фиксирует запрос субъекта (заёмщика) на
получение кредита в рамках конкретной **Кредитной программы** (раздел 02): запрашиваемую сумму, срок,
метод погашения, льготные периоды, документы заявителя, кредитные и залоговые документы, проектные
документы. Заявка проходит визуально отображаемый жизненный цикл (Подана → На рассмотрении → Одобрена →
Регистрация кредита) и направляется на рассмотрение двух независимых комиссий — кредитной и залоговой.

**Комиссия по заявкам (Проверка комиссии)** — запись голосования коллегиального органа по конкретной
заявке. Содержит состав комиссии (члены + председатель), индивидуальные решения голосования
(Одобрить / Отклонить / Воздержаться), прогресс голосования (тэлли N/M), крайние сроки для каждого
члена, протокол (номер + дата) и финальное решение, устанавливаемое председателем вручную.

Одна заявка может иметь несколько записей комиссий (кредитная и залоговая работают параллельно
и независимо). Тип документа в комиссионной записи («Заявки» / «Залог») указывает, к какому треку
относится рассмотрение.

## 3. Поля

### 3.1 Диалог создания заявки «Новая заявка»

Открывается кнопкой «Создать» из списка `/loan-applications`. URL страницы не меняется.
Кнопки диалога: **Далее** · **Отмена**.

#### Блок «Данные заявителя»

| Поле | Тип | Обяз. | Справочник | Описание |
|---|---|---|---|---|
| Субъект | `lookup` (`jmix-value-picker`) | Нет* | Субъекты | Заёмщик; адрес заполняется автоматически из записи субъекта |
| Адрес | `text` (только для чтения) | — | — | Автоматически подставляется из выбранного субъекта |
| Номер телефона | `text` | Нет | — | Контактный телефон; маска `+996 _ _ _ _ _ _ _ _ _` |

#### Блок «Условия кредита»

| Поле | Тип | Обяз. | Справочник | Описание |
|---|---|---|---|---|
| Кредитная программа | `lookup` (`jmix-value-picker`) | Нет* | Кредитные программы | Программа, в рамках которой подаётся заявка |
| Источник финансирования | `lookup` (`jmix-value-picker`) | Нет | Источник финансирования | Источник средств |
| Дополнительная информация | `textarea` | Нет | — | Произвольный комментарий |
| Запрашиваемая сумма | `decimal` (`vaadin-big-decimal-field`) | **Да** | — | Сумма запроса; поле обязательное, подсвечивается красным сразу при открытии диалога |
| Запрашиваемый срок | `integer` (`vaadin-integer-field`) | **Да** | — | Срок в месяцах; поле обязательное, подсвечивается красным сразу при открытии диалога |
| Метод погашения кредита | `select` | Нет | — | Схема погашения: **Аннуитетный** (по умолчанию) / **Дифференцированный** |

#### Блок «Льготный период» (под секцией «Ручные параметры льготного периода»)

| Поле | Тип | Обяз. | Справочник | Описание |
|---|---|---|---|---|

| Льготный период по начислению процентов | `checkbox` | Нет | — | По умолчанию выключен |
| Льготный период по процентам | `checkbox` | Нет | — | По умолчанию выключен |

> \* «Субъект» и «Кредитная программа» не помечены обязательными (`required=false`) на форме;
> заявка может быть создана без них.

## 4. Функции и действия

### 4.1 Кнопки панели действий (список заявок)

| Кнопка | Доступность | Описание |
|---|---|---|
| **Добавить условие поиска** | Всегда | Открывает конструктор фильтра |

> блокируются при отсутствии выделенной строки. При наличии выделения — доступны на любом статусе.
> Проверено 2026-06-19, `requirements/features/03-application-commission.md`.

### 4.2 Фильтр и пагинация (список заявок)

Список открывается с предустановленным фильтром (генерируется из строки параметров URL
`genericFilterCondition=...`). Конструктор фильтра предоставляет условия по полям:
**number** (Номер) · **status** (Статус) · **loanProgram** (Кредитная программа) · **createdDate** (Дата создания).
Пагинация: на тест-стенде отображается 21 строка. Проверено 2026-06-23,
`scripts/inspect/tz/dump.mjs loan-applications`.

### 4.3 Кнопки панели действий (список комиссий)

| Кнопка | Доступность | Описание |
|---|---|---|
| **Создать** | Всегда | Открывает форму создания новой записи комиссии |
| **Изменить** | При выборе строки | Открывает модальный диалог редактирования |

- Вкладка «Проектные документы» — кнопка **«Исключить»**.

## 5. Бизнес-правила и расчёты

### 5.1 Обязательные поля при создании

При попытке нажать «Далее» в диалоге «Новая заявка» без заполнения **«Запрашиваемая сумма»** и
**«Запрашиваемый срок»** диалог не переходит на следующий шаг: поля подсвечиваются красным с
сообщением «Поле является обязательным». Причём красная подсветка появляется сразу при открытии
диалога, до любого взаимодействия пользователя.
Проверено 2026-06-19, `requirements/features/03-application-commission.md`.

### 5.2 Автоматическое заполнение «Адрес»

При выборе субъекта в диалоге создания поле «Адрес» заполняется автоматически из записи субъекта
и остаётся доступным только для чтения. Проверено 2026-06-19,
`requirements/features/03-application-commission.md`.

### 5.3 Значения метода погашения

Select «Метод погашения кредита» принимает два значения: **Аннуитетный** (по умолчанию при создании) /
**Дифференцированный**. Проверено 2026-06-23, `scripts/inspect/tz/probe-app-create.mjs` (текст диалога).

### 5.4 График платежей на уровне заявки (вкладка «График»)

График строится для одобренного кредита по условиям, одобренным комиссией. Пример из записи
`/loan-applications/28` («Заявка - 60», сумма 200 000 KGS, срок 12 мес., метод аннуитетный,
ставка 12,00%, периодичность ежемесячно, проверено 2026-06-23,
`scripts/inspect/tz/probe-app-fields.mjs`):

| Колонка | Описание |
|---|---|
| № платежа | Порядковый номер |
| Дата платежа | Дата очередного платежа (напр. «10.04.2026 00:00», «11.05.2026 00:00») |
| Сумма платежа | Общая сумма платежа в периоде (напр. 16 666,67 KGS) |
| Основной долг | Часть платежа, идущая в погашение основного долга (напр. 16 666,67 KGS) |
| Проценты | Часть платежа, идущая в погашение процентов (напр. 0,00 — в льготный период) |
| Остаток до платежа | Остаток основного долга до платежа (напр. 200 000,00) |
| Остаток после платежа | Остаток основного долга после платежа (напр. 183 333,33) |

На тест-стенде для записи 28 первый платёж: 16 666,67 / 16 666,67 / 0,00 / 200 000,00 / 183 333,33.

### 5.5 Механика голосования комиссии

Прогресс голосования отображается в виде тэлли N/M в колонке «Комиссия» списка комиссий (напр. «1/4»,
«0/4», «2/4», «1/3»). M — количество членов комиссии; N — количество проголосовавших.

Финальное решение («Тип решения») устанавливается вручную председателем; голоса членов —
совещательные. Минимальный кворум не определён: председатель закрывает голосование в любой момент.
Кредитная и залоговая комиссии работают параллельно и независимо — рассмотрение одной не блокирует
другую. Подтверждено кастомером 2026-06-19,
`requirements/features/03-application-commission.md`.

### 5.6 Секция «Условия кредита, одобренные комиссией»

После вынесения решения кредитной комиссией секция на вкладке «Общая информация» заполняется
одобренными условиями: сумма, срок, ставка, периодичность, метод погашения, льготные периоды,
день платежа, источник финансирования, комментарий. Для каждого параметра суммы и срока выводятся
«Доступные значения» из кредитной программы (напр. «Доступные значения: 500000.00, 200000.00»).
Проверено 2026-06-23, `scripts/inspect/tz/probe-app-general-tab.mjs` (запись `/loan-applications/28`).

## 7. Связи

| Направление | Сущность | Кратность | Описание |
|---|---|---|---|
| Заявка → Госрешение | [01. Госрешение](01-gosreshenie.md) | N:1 | Через кредитную программу: каждая заявка ссылается на программу, программа — на госрешение |
| Заявка → Кредитная программа | [02. Кредитная программа](02-kreditnaya-programma.md) | N:1 | Каждая заявка создаётся в рамках одной программы; программа задаёт допустимые суммы, сроки, ставки. Подтверждено: колонка «Кредитная программа» в списке |
| Заявка → Субъект (заёмщик) | Субъект (Раздел «Заемщик») | N:1 | Заявка привязана к одному субъекту через поле «Субъект» |
| Заявка ← Комиссия по заявкам | Запись комиссии (`/loan-application-commissions`) | 1:N | Одна заявка может иметь несколько комиссионных записей (кредитная + залоговая, а также повторные рассмотрения). Связь подтверждена: колонка «Номер документа» в списке комиссий + кнопка «Просмотр заявки» в детали комиссии |
| Заявка → Кредит | Кредит | 1:0..1 | При достижении шага «Регистрация кредита» заявка становится основанием для регистрации кредитного договора. Переход фиксируется в стeppере; конкретный маршрут регистрации кредита принадлежит разделу 05 (Кредит) |
| Комиссия → Комиссия (орган) | Справочник «Комиссия» | N:1 | Каждая запись комиссии ссылается на конкретный коллегиальный орган (напр. «Газификация-Комиссия») |
| Комиссия → Члены | Справочник «Сотрудники» | N:M | Состав комиссии: роль (Член / Председатель), сотрудник, индивидуальное решение, крайний срок, дата отзыва, комментарий |

> Связь «Заявка → Кредитная программа» подтверждена на тест-стенде: все 21 запись в списке заявок
> (кроме нескольких без программы) содержат заполненную колонку «Кредитная программа».
> Проверено 2026-06-23, `scripts/inspect/tz/dump.mjs loan-applications`.
>
> Связь «Заявка ← Комиссия» подтверждена: список комиссий содержит ссылку на «Номер документа»
> (Заявка-28, Заявка-30, Заявка-43 и др.); при просмотре записи комиссии доступна кнопка
> «Просмотр заявки». Проверено 2026-06-23, `scripts/inspect/tz/probe-app-fields.mjs`.
