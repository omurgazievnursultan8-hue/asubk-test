## 1. Назначение

Кредитная программа — **шаблон кредитного продукта**, определяющий все параметры, на
основе которых создаются кредитные заявки и кредиты. Программа ссылается ровно на одно
Решение правительства (раздел 01) и является родительской сущностью для кредитных
заявок (раздел 03) и кредитов.

Программа задаёт: наименование и правовое основание, назначение и вид кредита, сроки
действия, параметры суммы и срока кредита (фиксированные или диапазонные), процентную
и штрафные ставки, льготные периоды, параметры графика платежей, перечни документов и
требования к залоговому обеспечению. Новая программа создаётся в статусе «Черновик» и
переводится в статус рассмотрения через действие «Отправить на подтверждение» на
девятой вкладке формы.

## 3. Поля

Поля сгруппированы по вкладкам мастера создания. Обозначения типов:

- `text` — однострочное текстовое поле (`vaadin-text-field`)
- `textarea` — многострочное текстовое поле (`vaadin-text-area`)
- `date` — поле выбора даты (`vaadin-date-picker`)
- `select` — выпадающий список (`vaadin-select`); значения перечислены в разделе 5
- `lookup` — поле выбора из справочника (`jmix-value-picker`)
- `multi-lookup` — множественный выбор из справочника (`jmix-multi-value-picker`)
- `combo` — комбо с поиском (`vaadin-combo-box`)
- `multi-combo` — комбо с множественным выбором (`vaadin-multi-select-combo-box`)
- `int` — целочисленное поле (`vaadin-integer-field`)
- `decimal` — поле с десятичным числом (`vaadin-big-decimal-field`)
- `checkbox` — флажок (`vaadin-checkbox`)
- `grid` — редактируемая таблица с кнопками «Добавить» / «Удалить» / «Исключить»
- `readonly` — поле только для чтения

Обязательность в колонке «Обяз.»: **Да** — проверено валидацией при сохранении
(кнопка «OK»; незаполненные поля выделяются красным, выводится toast «Заполните все
обязательные поля!»); **Нет** — необязательное при сохранении; **readonly** — выводится
системой, ввод недоступен.

Кнопки: **OK** · **Сохранить** · **Отмена** · **«Отправить на подтверждение»** (только на этой вкладке).

## 5. Бизнес-правила и расчёты

### 5.1 Обязательность при сохранении

При нажатии **«OK»** или **«Сохранить»** система проверяет **7 обязательных полей вкладки 1**:

1. Наименование на русском
2. Краткое описание
3. Решение правительства
4. Назначение кредита
5. Вид кредита
6. Дата с
7. Дата по

При незаполненном хотя бы одном из них выводится toast «Внимание / Заполните все
обязательные поля!» и незаполненные поля выделяются красным. Запись не создаётся.
Проверено 2026-06-17, `requirements/features/02-loan-program.md`.

Поля вкладок 2–8 (сумма, срок, ставки, штрафы, льготный период, платежи, документы,
залог) **необязательны при сохранении черновика** — программу можно сохранить без них.

### 5.2 Параметры суммы кредита

- **Тип «Фиксированная»** (значение «1»): грид «Фиксированные суммы» с кнопками
  «Добавить» / «Удалить» доступен для ввода конкретных сумм.
- **Тип «Диапазон»** (значение «2»): при выборе этого значения грид скрывается и
  вместо него появляются два поля `vaadin-big-decimal-field`:
  **«Минимальная сумма кредита»** и **«Максимальная сумма кредита»**.
  На тест-стенде все 9 программ используют тип «Фиксированная» с незаполненным
  гридом; поля диапазона сумм значений не содержат
  (проверено 2026-06-23, `scripts/inspect/tz/probe-diapazon-and-values.mjs`).
- Валюта суммы наследуется с вкладки 1 и отображается как read-only
  (на тест-стенде: KGS для программы ID 13 «тест1»,
  проверено 2026-06-23, `scripts/inspect/tz/probe-edit-values.mjs`).

### 5.3 Параметры срока кредита

- **Тип «Фиксированная»** (значение «1»): грид «Фиксированные сроки» с кнопками
  «Добавить» / «Удалить» доступен для ввода конкретных сроков.
- **Тип «Диапазон»** (значение «2»): при выборе этого значения грид скрывается и
  вместо него появляются два поля `vaadin-integer-field`:
  **«Минимальный срок»** и **«Максимальный срок»**.
  На тест-стенде все 9 программ используют тип «Фиксированная» с незаполненным
  гридом; поля диапазона сроков значений не содержат
  (проверено 2026-06-23, `scripts/inspect/tz/probe-srok-diapazon.mjs`).

Структура формы зафиксирована инспекцией 2026-06-17,
`requirements/features/02-loan-program.md`; Диапазон-рендеринг подтверждён
вживую 2026-06-23, `scripts/inspect/tz/probe-srok-diapazon.mjs`.

### 5.4 Параметры процентной ставки

- **Тип «Фиксированная»** (значение «1»): значение(я) выбираются из
  регламентированного справочника ставок (`vaadin-multi-select-combo-box`).
- **Тип «Диапазон»** (значение «2»): при выборе этого значения поле справочника
  скрывается и вместо него появляются два поля `vaadin-big-decimal-field`:
  **«Значение ставки с»** и **«Значение ставки по»**.
  На тест-стенде: программа «тест1» (ID 13) — ставка 6,00–12,00%;
  программа «Газификация районов» (ID 6) — ставка 2,00–11,00%
  (проверено 2026-06-23, `scripts/inspect/tz/probe-edit-values.mjs`).
- Флаг «Использовать плавающую ставку» — чекбокс; на вкладке 3 дополнительных
  полей плавающей ставки не отображается
  (зафиксировано 2026-06-17, `requirements/features/02-loan-program.md`;
  подтверждено 2026-06-23, `scripts/inspect/tz/dump-loan-program-tabs.mjs`).

Тип контрола `vaadin-multi-select-combo-box` подтверждён 2026-06-23,
`scripts/inspect/tz/dump-loan-program-tabs.mjs`.

### 5.5 Параметры штрафных ставок

- **Штраф по основной сумме**: тип (Фиксированная / Диапазон) + числовое значение.
- **Ограничение штрафа по процентам**: предельный размер штрафа (% от суммы).
- **Штраф по процентам**: тип (Фиксированная / Диапазон) + числовое значение.

Все поля — десятичные числа (`vaadin-big-decimal-field`). Подтверждено 2026-06-23:

## 6. Статусная модель

```
  ┌──────────────┐    «Отправить на подтверждение»
  │   Черновик   │ ─────────────────────────────────────────► [ ? ]
  │  (создание)  │
  └──────────────┘
```

| Статус | Описание |
|---|---|
| Черновик | Начальный статус после создания. Программа редактируется через «Изменить», удаляется через «Удалить». Все 9 записей на тест-стенде находятся в этом статусе (проверено 2026-06-23, колонка «Статус» списка) |

> Переход из «Черновика» в следующий статус инициируется кнопкой
> **«Отправить на подтверждение»** на вкладке 9. Наименование следующего статуса
> на тест-стенде не верифицировано: записей в ином статусе не обнаружено
> (проверено 2026-06-23, `scripts/inspect/tz/dump.mjs loan-programs`);
> наименование «На рассмотрении» выводится из подписи кнопки и не подтверждено
> наблюдением [ ? ].
> Дальнейшие статусы и переходы (кто подтверждает, какие статусы следуют)
> из пользовательского интерфейса не установлены.

## 7. Связи

| Направление | Сущность | Кратность | Описание |
|---|---|---|---|
| Кредитная программа → Госрешение | [01. Госрешение](01-gosreshenie.md) | N:1 | Каждая программа ссылается ровно на одно госрешение («Решение правительства»); без него создание невозможно |
| Кредитная программа ← Кредитная заявка | 03. Кредитная заявка | 1:N | Каждая кредитная заявка ссылается на программу как на шаблон |
| Кредитная программа ← Кредит | Кредит | 1:N | Кредиты, выданные в рамках программы, ссылаются на неё |
| Кредитная программа → Назначение кредита | Справочник «Цели кредита» | N:1 | Цель использования средств |
| Кредитная программа → Вид кредита | Справочник «Вид кредита» | N:1 | Тип кредитного продукта |
| Кредитная программа → Отрасль | Справочник «Отрасли и направления» | N:1 | Сектор экономики заёмщика |
| Кредитная программа → Валюта | Справочник «Вид валют» | N:1 | Валюта программы |
| Кредитная программа → Источник финансирования | Справочник «Источник финансирования» | N:1 | Категория источника средств |
| Кредитная программа → Кредитная линия | Справочник «Кредитная линия» | N:1 | Конкретная кредитная линия / донор |

> Связь Программа → Госрешение подтверждена на тест-стенде: колонка «Решение
> правительства» в списке отображает наименование родительского решения
> (напр. «ПКМКР №554 от 11.09.2024») для каждой из 9 программ.
> Проверено 2026-06-23, `scripts/inspect/tz/dump.mjs loan-programs`.
