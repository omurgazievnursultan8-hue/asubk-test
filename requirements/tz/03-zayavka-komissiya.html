1. Назначение

Заявка (Кредитная заявка) — сущность кредитного цикла. Фиксирует запрос субъекта (заёмщика) на получение кредита в рамках конкретной Кредитной программы: запрашиваемую сумму, срок, метод погашения, льготные периоды, документы заявителя, кредитные и залоговые документы, проектные документы. Заявка проходит визуально отображаемый жизненный цикл (Подана → На рассмотрении → Одобрена → Регистрация кредита) и направляется на рассмотрение двух независимых комиссий — кредитной и залоговой.

Комиссия по заявкам (Проверка комиссии) — запись голосования коллегиального органа по конкретной заявке. Содержит состав комиссии (члены + председатель), индивидуальные решения голосования (Одобрить / Отклонить / Воздержаться), прогресс голосования (тэлли N/M), протокол (номер + дата) и финальное решение, устанавливаемое председателем вручную.

Одна заявка может иметь несколько записей комиссий (кредитная и залоговая работают параллельно и независимо). Тип документа в комиссионной записи («Заявки» / «Залог») указывает, к какому треку относится рассмотрение.

5. Бизнес-правила и расчёты

5.1 Обязательные поля при создании

При нажатии «Далее» без заполнения «Запрашиваемая сумма» и «Запрашиваемый срок» диалог не переходит дальше: поля в состоянии invalid=true, красная подсветка. Подсветка появляется сразу при открытии диалога, до взаимодействия. Навигация заблокирована до заполнения.

5.2 Автоматическое заполнение «Адрес»

При выборе субъекта в диалоге создания поле «Адрес» заполняется автоматически из записи субъекта и остаётся только для чтения.

5.3 Значения метода погашения

Select «Метод погашения кредита»: Аннуитетный (по умолчанию) / Дифференцированный.

5.4 График платежей (вкладка «График»)

График строится для одобренного кредита по условиям, одобренным комиссией.

КолонкаОписание
№ платежаПорядковый номер
Дата платежаДата очередного платежа
Сумма платежаОбщая сумма платежа в периоде
Основной долгЧасть платежа в погашение основного долга
ПроцентыЧасть платежа в погашение процентов
Остаток до платежаОстаток основного долга до платежа
Остаток после платежаОстаток основного долга после платежа
Пример (Заявка-60, сумма 200 000 KGS, 12 мес., аннуитет, ставка 12%): первый платёж 16 666,67 / 16 666,67 / 0,00 / 200 000,00 / 183 333,33. Проценты = 0 во всех строках — дефект расчёта, проценты по ставке за период не начисляются.

5.5 Механика голосования комиссии

Прогресс голосования — тэлли N/M в колонке «Комиссия» (M — членов, N — проголосовало). Финальное решение («Тип решения») устанавливается вручную председателем; голоса членов совещательные. Минимальный кворум не определён: председатель закрывает голосование в любой момент. Кредитная и залоговая комиссии работают параллельно и независимо.

5.6 Секция «Условия, одобренные комиссией»

После решения кредитной комиссии секция на «Общей информации» заполняется одобренными условиями: сумма, срок, ставка, периодичность, метод погашения, льготные периоды, день платежа, источник финансирования, комментарий. Для суммы и срока выводятся «Доступные значения» из программы.

5.7 Генерация номера заявки

Как есть: «Номер документа» — сквозной счётчик вида «Заявка - 60» (голый int). Не сортируется между годами, не несёт контекста.

Целевой формат:

З-2026-000123

СегментЗначениеПравило
Зпрефикс «заявка»фиксированный
2026год присвоения4 цифры
000123порядковый номер6 цифр, ведущие нули

Правила:

  • Сброс серии — годовой: с 1 января счётчик снова с 000001. Ключ — год.
  • Момент присвоения — при подаче заявки (действие «Подать»), не в черновике. Черновик имеет временный технический id; официальный номер штампуется один раз при переходе в реестр.
  • Неизменяемость — присвоенный номер не редактируется и не переиспользуется.
  • Потокобезопасность — атомарный счётчик/sequence БД по ключу года. Запрещено max(id)+1 и клиентский счётчик (гонка → дубль). В Jmix — Sequences / idGenerator Numbers.
  • Без кода программы/органа в номере.
