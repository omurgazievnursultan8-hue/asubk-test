# 01. Госрешение

> Маршрут: `/gov-decisions`. Проверено 2026-06-23, `scripts/inspect/tz/dump.mjs gov-decisions`;
> форма создания — `scripts/inspect/tz/dump-form-alllabels.mjs` (маршрут `/gov-decisions/new`).

---

## 1. Назначение

Госрешение (Решение правительства) — **корневая сущность кредитной подсистемы**.
Фиксирует нормативно-правовое основание (постановление, распоряжение и т. д.),
которое разрешает государственное кредитование. Каждая кредитная программа
ссылается ровно на одно госрешение; без него создание программы невозможно.
Записи проходят процесс согласования: сначала рассматриваются, затем одобряются
или закрываются (при отклонении).

## 4. Функции и действия

### 4.1 Кнопки панели действий (список)

| Кнопка | Доступность | Описание |
|---|---|---|
| **Создать** | Всегда | Открывает страницу `/gov-decisions/new` |
| **Изменить** | При выборе строки; отключена для статусов «Одобрен» и «Закрыт» | Открывает модальный диалог редактирования |
| **Просмотр** | При выборе строки | Открывает модальный диалог просмотра (поля только для чтения) |
| **Удалить** | При выборе строки | Удаляет запись (физически); доступна на всех статусах |
| **Одобрить** | При выборе строки | Переводит запись в статус «Одобрен» немедленно, без подтверждения |
| **Отклонить** | При выборе строки | Переводит запись в статус «Закрыт» немедленно, без подтверждения и без запроса причины |

### 4.2 Дополнительные элементы управления (список)

- **Добавить условие поиска** — конструктор фильтра; по умолчанию список открывается с предустановленным (но удаляемым) фильтром по статусу «Одобрен».
- **Фильтр: фильтр по статусу** — метка активного фильтрующего условия, отображается в панели фильтрации.
- Настройки столбцов (шестерёнка) — видимость и порядок колонок (стандартный Jmix-компонент).
- Нумерация строк / пагинация (стандартный Jmix-компонент).

## 5. Бизнес-правила

1. **Обязательные поля:** при попытке сохранить форму без заполнения хотя бы одного из пяти обязательных полей (Наименование, Краткое наименование, Код, Номер решения, Вид решения) система отображает toast-уведомление «Внимание / Заполните все обязательные поля!» и подсвечивает незаполненные поля красным.

2. **Автоматический начальный статус:** при создании статус всегда устанавливается системой в «На стадии рассмотрения». Изменить его вручную через форму нельзя — поле «Статус» только для чтения.

3. **Автоматическое заполнение даты:** при открытии формы создания «Дата решения» автоматически заполняется текущей датой.

4. **Блокировка редактирования по статусу:** кнопка «Изменить» недоступна для записей в статусах «Одобрен» и «Закрыт».

5. **Одобрение и отклонение — только через список:** переход статуса осуществляется исключительно кнопками «Одобрить» / «Отклонить» в списке, не через форму.

6. **Одобрение** переводит запись в «Одобрен» без дополнительного подтверждения.

7. **Отклонение** переводит запись в «Закрыт» без подтверждения и без сохранения причины отклонения. Действие необратимо через UI.

8. **Удаление:** кнопка «Удалить» активна на любом статусе (включая «Одобрен» и «Закрыт»).

## 6. Статусная модель

```
                   ┌─────────────────────────────────┐
                   │    На стадии рассмотрения        │
                   │  (начальный, при создании)       │
                   └──────────────┬──────────────────┘
                                  │
                  ┌───────────────┴───────────────┐
                  │                               │
                  ▼                               ▼
         ┌──────────────┐               ┌──────────────┐
         │  Одобрен     │               │  Закрыт      │
         │ (терминал)   │               │ (терминал,   │
         │              │               │  при         │
         │ Изменить — ✗ │               │  отклонении) │
         │ Удалить  — ✓ │               │ Изменить — ✗ │
         └──────────────┘               │ Удалить  — ✓ │
                                        └──────────────┘
```

| Статус | Описание | Доступные действия |
|---|---|---|
| На стадии рассмотрения | Начальный статус после создания | Изменить, Удалить, Одобрить, Отклонить |
| Одобрен | Решение согласовано; на его основе можно создавать кредитные программы | Просмотр, Удалить |
| Закрыт | Терминальный статус: устанавливается при нажатии «Отклонить» | Просмотр, Удалить |

> Статус «Закрыт» является терминальным.

> Перечень статусов верифицирован 2026-06-23 путём сброса предустановленного фильтра «фильтр по статусу» и чтения колонки «Статус» по всем 13 записям списка (`scripts/inspect/tz/probe-gov-statuscol7.mjs`). Наблюдаемые значения: «На стадии рассмотрения», «Одобрен», «Закрыт».
