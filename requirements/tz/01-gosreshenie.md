## 1. Назначение

Госрешение (Решение правительства) — **корневая сущность кредитной подсистемы**.
Фиксирует нормативно-правовое основание (постановление, распоряжение и т. д.),
которое разрешает государственное кредитование. Каждая кредитная программа
ссылается ровно на одно госрешение; без действующего госрешения создание
программы невозможно.

Запись проходит управляемый жизненный цикл согласования: создаётся автором,
отправляется на рассмотрение, согласующий одобряет / отклоняет / возвращает на
доработку; одобренное решение вступает в силу («Действует») и может быть позднее
закрыто. Каждый переход статуса фиксируется в журнале аудита.

## 3. Поля

### 3.1 Форма создания / редактирования

| Поле | Тип | Обяз. | Справочник | Описание |
|---|---|---|---|---|
| Наименование | Текстовое поле | Да | — | Полное официальное наименование решения |
| Краткое наименование | Текстовое поле | Да | — | Краткое название, отображается в списке и в связанных программах |
| Код | Текстовое поле, **только для чтения** | Авто | — | Генерируется системой при создании (R3); ручной ввод убран |
| Номер решения | Текстовое поле | Да | — | Официальный номер нормативного акта (напр. «80-р», «№554») |
| Вид решения | Поле выбора из справочника | Да | Вид решения | Тип акта: Распоряжение, Постановление, Основное, РКМКР, РПКР, ПКМКР |
| Дата решения | Дата | Да | — | Дата принятия акта; по умолчанию — текущая дата; **запрет будущих дат** (R5) |
| Статус | Текстовое поле, только для чтения | Авто | — | Устанавливается системой по жизненному циклу; при создании — «Черновик» |
| Действующее решение | Бейдж Да/Нет | — | — | **Отдельный хранимый флаг**, не выводится из статуса (R7); семантика — см. §10 (открытые вопросы) |
| Примечание | Многострочный текст (вкладка «Примечание») | Нет | — | Произвольный комментарий |

> **Кнопки формы (режим редактирования):** OK (сохранить) · Отмена.

### 3.2 Код решения — генерация (R3)

- Код **генерируется системой** при создании, поле только для чтения, ручной ввод исключён.
- Формат: `{ТИП}-{ГГГГ}-{NNNN}`, где
  - `ТИП` — короткий код вида решения: `ПОСТ` / `РАСП` / `ОСН` / `РКМКР` / `РПКР` / `ПКМКР`;
  - `ГГГГ` — год из «Даты решения»;
  - `NNNN` — сквозной порядковый номер внутри пары «тип + год», с ведущими нулями.
- Пример: `ПОСТ-2026-0007`.
- Уникальность гарантируется счётчиком на стороне сервера.
- Старые пустые legacy-коды миграции не требуют (предстоит полное обнуление данных).

| Колонка | Описание |

| Наименование | Полное наименование |
| Краткое наименование | Краткое наименование |

| Код | Системный код записи |
| Номер решения | Официальный номер акта |
| Вид решения | Тип акта |

## 4. Страница деталей (R6)

Режим **«Просмотр + действия»**: данные показываются как «подпись → значение»,
без редактируемых инпутов; редактирование — отдельным режимом по кнопке «Изменить».

Компоненты страницы:

- **Слева** — дерево-навигация модуля (Кредиты → Программы и решения → Решение правительства).
- **Шапка** — хлебные крошки + наименование/номер решения + бейдж статуса.
- **Вкладки:**
  - **Общая информация** — режим просмотра: Наименование, Краткое наименование, Статус, Код, Номер решения, Вид решения, Дата решения, «Действующее решение» (Да/Нет), Примечание. Для статуса «Отклонён» дополнительно показывается «Причина отклонения» (R2).
  - **Кредитные программы** — таблица связанных программ (downstream-ссылки, см. §8 и R4).
  - **Документы** — таблица вложений (R1).
  - **История** — таймлайн действий с иконками по типу (создание / изменение / загрузка файла / смена статуса); единый журнал аудита (R2).
- **Футер** — «Изменить» (вход в режим редактирования) + «Назад». В режиме редактирования поля становятся инпутами, футер — `OK / Отмена`.

Действия жизненного цикла доступны **и в списке, и на странице деталей**; видимость
кнопок — по роли, доступность — по статусу (см. §6, §7).

## 5. Документы решения (R1)

- Отдельный блок загрузки файлов, **независимый от «Примечания»**.
- Формат — **только PDF**; лимит размера ≈ 10–20 МБ; валидация при загрузке.
- Можно прикрепить **несколько** файлов (основное решение + приложения/изменения);
  ровно один помечается как **основной**.
- На статусах «Черновик» / «На рассмотрении» файл **необязателен**; для перехода в
  «**Действует**» наличие основного документа **обязательно** (увязано с §6).
- Замена файла — загрузкой **новой версии** с сохранением истории (без перезаписи).
- У действующего решения удаление файла запрещено.
- Кто и когда загрузил/заменил файл — фиксируется в журнале аудита (R2).

## 6. Функции и действия

### 6.1 Действия жизненного цикла

Доступны из списка и со страницы деталей. Видимость — по роли; доступность — по статусу.

| Действие | Из статуса | В статус | Роль |
|---|---|---|---|
| Создать | — | Черновик | Автор |
| Отправить | Черновик | На рассмотрении | Автор |
| Отозвать | На рассмотрении | Черновик | Автор |
| Отправить повторно | На доработке | На рассмотрении | Автор |
| Удалить | Черновик | (удалено, soft-delete) | Автор |
| Одобрить | На рассмотрении | Действует | Согласующий |
| Отклонить | На рассмотрении | Отклонён | Согласующий |
| Вернуть на доработку | На рассмотрении | На доработке | Согласующий |
| Закрыть | Действует | Закрыт | Согласующий / уполномоченный |
| Вернуть на рассмотрение | Закрыт | На рассмотрении | Администратор |
| Вернуть на рассмотрение | Отклонён | На рассмотрении | Администратор |

> «Создать» не зависит от статуса — доступно роли Автор всегда (создаёт новый «Черновик»).
> Два «Вернуть на рассмотрение» — служебный админ-возврат при мис-клике (см. §7).

### 6.2 Прочие действия

| Кнопка | Доступность | Описание |
|---|---|---|
| **Просмотр** | При выборе строки | Открывает страницу деталей (только чтение) |
| **Изменить** | Только в статусах «Черновик» / «На доработке» | Включает режим редактирования |

### 6.3 Подтверждения и причины

- **Отклонить** (R2): диалог подтверждения с **обязательным** полем «Причина отклонения»;
  без причины действие недоступно. Причина видна в просмотре записи и в списке.
- **Закрыть**, **Вернуть на доработку**, **Вернуть на рассмотрение** (админ),
  **Удалить** — через диалог подтверждения; обязательность комментария-причины —
  см. §10 (открытые вопросы).

### 6.4 Дополнительные элементы управления (список)

- **Добавить условие поиска** — конструктор фильтра.
- **Фильтр по статусу** — предустановленный, но снимаемый фильтр в панели фильтрации.
- Настройки столбцов (видимость/порядок), нумерация строк / пагинация — стандартные Jmix-компоненты.

## 7. Роли и матрица прав (R7)

| Роль | Назначение |
|---|---|
| **Автор** | Создаёт и редактирует решения, отправляет на согласование, дорабатывает |
| **Согласующий** | Принимает решение по согласованию: одобряет, отклоняет, возвращает, закрывает |
| **Администратор** | Служебный возврат «Закрыт» / «Отклонён» → «На рассмотрении» с причиной в журнал |
| **Наблюдатель** *(опц.)* | Только просмотр; ввод роли — см. §10 (открытые вопросы) |

**Матрица прав** (по статусу):

| Статус | Автор | Согласующий |
|---|---|---|
| Черновик | Изменить · Удалить · Отправить | только просмотр |
| На рассмотрении | Отозвать | Одобрить · Отклонить · Вернуть на доработку |
| На доработке | Изменить · Отправить повторно | только просмотр |
| Действует | только просмотр | Закрыть |
| Отклонён | только просмотр | только просмотр |
| Закрыт | только просмотр | только просмотр |

> Тулбар: **скрывать кнопки по роли, гасить по статусу** (с подсказкой на неактивных).
> Возврат из «Закрыт» / «Отклонён» — вне матрицы Автор/Согласующий, выполняется Администратором.

## 8. Бизнес-правила

1. **Обязательные поля:** при сохранении без любого из обязательных полей
   (Наименование, Краткое наименование, Номер решения, Вид решения, Дата решения)
   система выводит уведомление «Заполните все обязательные поля!» и подсвечивает
   незаполненные поля. «Код» не вводится вручную (§3.2).

2. **Системный код** (R3): генерируется при создании, поле только для чтения.

3. **Дата решения** (R5): по умолчанию — текущая дата; **будущие даты запрещены**
   (валидация на клиенте — календарь не даёт выбрать будущее, и на сервере — чтобы
   не обойти через API); при попытке — сообщение «Дата решения не может быть в будущем».

4. **Начальный статус** — «Черновик»; статус меняется только действиями жизненного
   цикла (§6.1), не ручной правкой формы.

5. **Редактирование** содержимого возможно только в статусах «Черновик» и «На доработке».

6. **Удаление** (R4) возможно только в статусе «Черновик»; soft-delete вместо
   физического (запись помечается удалённой, остаётся в БД, действие пишется в журнал);
   обязательная причина в модалке подтверждения. Серверная проверка ссылок: удаление
   запрещено, если на решение ссылается хотя бы одна программа (или иная downstream-сущность),
   с внятным сообщением («Нельзя удалить: на решение ссылаются N программ»).

7. **Переход в «Действует»** требует наличия основного документа (R1, §5).

8. **Отклонение** (R2): подтверждение + обязательная причина; запись в журнал аудита;
   ведёт в статус «Отклонён» (не «Закрыт»).

9. **Закрытие** доступно только из «Действует»; «Закрыт» означает отмену
   действовавшего решения (не отказ в согласовании).

10. **Обратимость** (R2/R7): «Закрыт» и «Отклонён» **не терминальны** — Администратор
    возвращает запись в «На рассмотрении» с обязательной причиной в журнал. Других
    выходов из этих статусов нет. Модели «Дублировать» нет — возврат покрывает оживление записи.

11. **Журнал аудита** (R2): каждый переход статуса, загрузка/замена файла, удаление —
    фиксируются (кто, когда, действие, причина). Это единый журнал, отображаемый на
    вкладке «История» (R6) — не дублировать отдельными сущностями.

## 9. Статусная модель (R7)

```
                 ┌───────────┐  Создать
                 │ Черновик  │◄───────────────── (новая запись)
                 └─────┬─────┘
          Отправить    │   ▲  Отозвать
                       ▼   │
                 ┌──────────────────┐
                 │ На рассмотрении  │◄──── Отправить повторно ──┐
                 └───┬────┬─────┬───┘                           │
        Одобрить     │    │     │  Вернуть                ┌──────┴──────┐
                     ▼    │     └────────────────────────►│ На доработке │
              ┌──────────┐│  Отклонить                    └──────────────┘
              │ Действует ││─────────────► ┌──────────┐
              └─────┬─────┘                │ Отклонён │
        Закрыть     │                      └────┬─────┘
                    ▼                           │ Вернуть на рассмотрение (админ)
              ┌──────────┐                      │
              │  Закрыт  │──────────────────────┘
              └────┬─────┘  Вернуть на рассмотрение (админ) → На рассмотрении
                   └──────────────────────────────────────►
```

| Статус | Описание | Доступные действия |
|---|---|---|
| Черновик | Создано и редактируется автором, ещё не отправлено | Изменить, Удалить, Отправить |
| На рассмотрении | Отправлено согласующему, ожидает решения | Отозвать; Одобрить, Отклонить, Вернуть на доработку |
| На доработке | Возвращено автору с замечаниями | Изменить, Отправить повторно |
| Действует | Одобрено и вступило в силу | Закрыть; просмотр |
| Отклонён | Отказ в согласовании (из «На рассмотрении») | Возврат на рассмотрение (админ); просмотр |
| Закрыт | Действовавшее решение отменено/утратило силу (из «Действует») | Возврат на рассмотрение (админ); просмотр |

> **Миграция данных:** записи живого стенда со статусом «Закрыт», означающие отказ в
> согласовании (прежний единый «Закрыт» = отказ), перевести в «Отклонён». «Закрыт»
> в целевой модели — только отмена действовавшего решения.

## 10. Открытые вопросы (требуют решения заказчика)

1. **Обязательность комментария-причины** при действиях «Отклонить» и «Вернуть на доработку».
   (Для «Отклонить» причина уже принята как обязательная — R2; для «Вернуть» — не зафиксировано.)
2. **Роль «Наблюдатель»** (только просмотр) — вводить ли её.
3. **Кто уполномочен закрывать решение** («Закрыть»: Согласующий / отдельная уполномоченная роль).
4. **Семантика флага «Действующее решение» (Да/Нет)** при статусе «Действует»: кто ставит,
   значение по умолчанию, что означает «Нет» при статусе «Действует». Флаг хранится
   отдельно и **не выводится** из статуса (R7).
