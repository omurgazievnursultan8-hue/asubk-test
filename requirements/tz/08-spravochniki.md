## 1. Назначение и границы

**Справочник** — экран управления нормативным перечнем значений (валюты, виды
кредита, статусы, отрасли и т. д.), на который ссылаются lookup-поля
(`jmix-value-picker`) и select-поля форм кредитного модуля. Из формы справочник
открывается как picker-диалог; из левого меню «Справочники» — как самостоятельный
экран-список с CRUD.

**Границы.** В разбор включены **47 справочников**, на которые ссылаются сущности
разделов 01–07 (Госрешение → … → Обслуживание). За границей остались справочники
внешних модулей (субсидии/СУГС, корпоративное управление, поручительства, взыскание
задолженности, кураторство, гео-справочники, безопасность) — они документируются
отдельно (`requirements/overview.md`).

> **Три справочника формы программы не найдены как самостоятельный пункт меню**
> (nav-инспекция 2026-06-24): **«Цели кредита»** (поле «Назначение кредита»),
> **«Уровни проверки документов»** и **«Типы расчета процентов»** — открываются
> только через picker внутри формы. Управляющего экрана в меню «Справочники» нет →
> непоследовательная поверхность администрирования (🟡, см. §7).

## 3. Архетипы экрана

Инспекция выявила **два устойчивых шаблона** управляющего экрана.

### 3.1 `S` — простой словарь (40 из 47)

- **Колонки:** одна-две — обычно `Наименование` (иногда доменная: `Процент`,
  `Период(кол-во мес.)` / `Тип периода(мес.)`, `Код` + `Наименование`).
- **Тулбар:** `Добавить условие поиска` · **Создать** · **Изменить** · **Удалить**.
  **Без «Просмотр».**
- **Редактор** — модальное окно с минимальным набором полей.
- **Пагинация** — стандартная Jmix снизу; конструктор фильтра доступен, но для
  словаря из 3–8 строк практически не нужен.

### 3.2 `R` — расширенный классификатор (7 из 47)

`industryDirections`, `collateralTypes`, `commissions`, `creditTerms`,
`calculationCoefficients`, `individuals`, `organizations`.

- **Колонки:** `Наименование` · **`Код`** · **`Статус`** · **`Дата создания`**
  (у реестров субъектов — доменные: ИНН/ФИО/паспорт, ИНН/ОКПО/адрес).
- **Тулбар:** добавляется **«Просмотр»** (read-only страница) — единственные
  экраны, где запись можно открыть, не входя в редактирование.
- Это более новый/стандартизованный шаблон сущности (код + статус + аудит-дата).

> **Вывод:** платформа уже содержит «правильный» шаблон (`R` с «Просмотр» и
> статусом), но он применён лишь к 7 экранам. Остальные 40 — голый CRUD без
> просмотра. Разнобой архетипов — основной структурный дефект слоя справочников.

## 4. Как работает справочник

1. **Из левого меню** — экран-список выбранного архетипа (§3).
2. **CRUD** — Создать/Изменить/Удалить через тулбар; редактор — модальное окно.
   На `S`-экранах открыть запись «только посмотреть» нельзя — лишь «Изменить».
3. **Фильтр** — конструктор «Добавить условие поиска» (jmix-filter) присутствует
   на всех экранах; пагинация — снизу.
4. **Из формы (picker-режим)** — поле `jmix-value-picker` (кнопка •••) открывает
   **полноэкранный справочник-диалог** со списком значений. Захваченные значения
   диалогов — в `02-loan-program-field-analysis.md` §«Захваченные справочники».
   - **Дефект picker-режима (P2-R9, проверено 2026-06-19):** picker «Решение
     правительства» открывает экран с действиями **Создать/Изменить/Удалить/
     Одобрить/Отклонить** — слишком широкий доступ из формы программы; фильтр
     статуса — съёмный UI-чип, серверной проверки нет. Должно быть: **режим
     только выбора** + жёсткий фильтр загрузчика. В мокапе
     `mockups/loan-program/loan-program.html` picker заблокирован на выбор.

## 7. Находки (severity-tagged)

- 🔴 **`loan-states` («Статус кредита») содержит мусор «Кредит компании»** —
  это не статус. Справочник статусов кредита фактически не наполнен валидными
  значениями жизненного цикла. Риск для раздела 05.
- 🔴 **Picker открывает полный CRUD с Одобрить/Отклонить** (P2-R9) — широкий
  доступ из формы; нужна блокировка на режим выбора + серверный фильтр.
- 🟠 **13 справочников пусты** (`order-document-types`, `entity-document-states`,
  `document-package-*`, `applied-entity-*`, `order-term-frequency-types`,
  `order-term-rate-periods`, `order-term-accr-methods`, `work-sectors`,
  `good-types`, `loan-payment-capacity-groups`). Часть из них — статус/тип-енумы,
  на которые ссылается воркфлоу; пустой справочник = поле без значений для выбора.
- 🟠 **Англоязычные заголовки колонок** — `agreement-templates`
  («AgreementTemplate.templateType», «…status», «…active») и
  `agreement-template-codes` («AgreementTemplateCode.code» …) показывают сырые
  Java-property-пути вместо локализованных подписей.
- 🟠 **Англоязычный заголовок экрана** — «Destination accounts» (`destination-accounts`)
  не локализован.
- 🟠 **40/47 экранов без режима «Просмотр»** — инспекция/аудит значения вынуждает
  входить в редактирование (риск случайной правки), как у программы (P2-R4).
- 🟡 **Статусы не отрисованы бейджами** — ни в справочниках-статусах, ни в
  колонке «Статус» расширенных классификаторов; дизайн-система предусматривает
  цветные бейджи.
- 🟡 **Тест-мусор в боевых-кандидатах** — «соновное 1» (опечатка, вид решения),
  «123/321» (счета погашения), «hidshgiu/fgxdzxh» (группа заёмщика), «шрафы»
  (опечатка, очередь погашения), «Aibek rate» (вид ставок).
