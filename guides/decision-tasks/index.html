Назначение пакета

Это набор документов по задачам улучшения раздела «Решение правительства» (первая фаза кредитного цикла). По каждой задаче — отдельный документ в едином формате. Порядок работы:

  1. Обсуждение с профильными сотрудниками по каждому документу.
  2. Согласование — утверждение задачи профильными сотрудниками.
  3. Передача утверждённых задач команде разработки.
Приоритеты: 🔴 Высокий 🟠 Средний 🟢 Низкий. Источник задач — TODO.md → Фаза 1 и requirements/features/01-government-decision.md.

Документы пакета

R1 · 🔴 Высокий

Вложение документа решения

Загрузка PDF отдельно от «Примечания»: несколько файлов, один основной, версии, обязателен для перехода в «Действует».

R2 · 🔴 Высокий

Отклонение: подтверждение и аудит

Диалог подтверждения, обязательная причина, журнал переходов, обратимость отклонения.

R3 · 🔴 Высокий

Системная генерация «Кода»

Код по формату {ТИП}-{ГГГГ}-{NNNN}, read-only, уникальность по счётчику.

R4 · 🟠 Средний

Защита удаления решений

Серверная проверка ссылок, блок по статусу, soft-delete с аудитом.

R5 · 🟠 Средний

Валидация «Даты решения»

Запрет будущих дат на клиенте и сервере, понятное сообщение об ошибке.

R6 · 🟠 Средний

Страница деталей по макету

Режим «Просмотр + действия», вкладки, бейдж статуса, история. Макет decision-detail.html.

R7 · 🔴 Высокий

Воркфлоу статусов

6 статусов, 10 переходов, роли, матрица прав. Спецификация decision-workflow.html.

Связки между задачами. Единый журнал аудита (R2) обслуживает вложения (R1), историю на странице деталей (R6) и переходы статусов (R7). Терминология «Действует» и модель обратимости согласуются в R7. Стиль полей формы — единый компонент дизайн-системы (R3, R6 и Фаза 2 / P2-R1).
